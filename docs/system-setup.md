## 구성 요소

### 제어기 본체

| 구성 | 사양 |
|------|------|
| MCU | Arduino Nano V3.0 (ATmega328P) |
| pH 센서 | DFRobot SEN0161-V2 + ADS1115 16bit ADC |
| 온도 센서 | DS18B20 (위즈 탱크 내 침수) |
| 통신 | HC-06 블루투스 (9600 baud) |
| 모터 드라이버 | L298N1/L298N2 (도징 펌프, 12V) + L298N3 (에어 펌프, USB 5V) |
| 속도 제어 | PWM 컨트롤러 Motor− 출력 → L298N ENA/ENB (역 PWM) |
| 전원 | 12V DC → Buck 5V (MCU) + USB 5V (L298N3 에어 펌프) |

### 연동 펌프 (4구, 모두 양방향)

| 펌프 | 경로 | 정방향 (f) | 역방향 (b) |
|------|------|------------|------------|
| 펌프1 (M1) | 수조 ↔ 수조물 비이커 | 수조 → 수조물 비이커 (샘플링) | 수조물 비이커 → 수조 (반환) |
| 펌프2 (M2) | 수조물 비이커 ↔ pH 측정 비이커 | 수조물 → pH 비이커 (측정 이송) | pH 비이커 → 수조물 (측정 후 반환) |
| 펌프3 (M3) | 3M KCl 비이커 ↔ pH 측정 비이커 | KCL → pH 비이커 (프로브 보관) | pH 비이커 → KCL 비이커 (저장수 배출) |
| 펌프4 (M4) | 위즈 탱크 ↔ pH 측정 비이커 | 참조수 → pH 비이커 (측정 이송) | pH 비이커 → 위즈 탱크 (참조수 반환) |

### 에어 공급 및 PWM 속도 제어 (L298N3)

| 핀 | 명령 (ON / OFF) | 용도 |
|-----|------|------|
| D12 (L298N3 채널A) | `ron` / `roff` | USB 5V 에어 펌프 ON/OFF (참조수 + 수조수 폭기) |
| D13 (L298N3 채널B) | `ton` / `toff` | PWM 속도 제어기 ON/OFF (모터 구동 시 활성화) |
| D12 + D13 | `airoff` | 둘 다 LOW (에어 OFF + PWM 제어기 OFF) |

- `ron`/`roff`: D12 → 에어 펌프 ON/OFF (참조수·수조수 동시 탈기)
- `ton`/`toff`: D13 → PWM 속도 제어기 ON/OFF (도징 펌프 구동 전 `ton`)
- `airoff`: D12·D13 동시 LOW (둘 다 OFF). 전체를 한 번에 끄려면 `stop`(모터 전체 + 에어/PWM 핀)
- **하드웨어 규칙:** 액체 이동(`mXf`/`mXb`) 직전에는 반드시 `airoff`로 폭기를 끈다.

> **참고:** 구 펌웨어의 `air:총초:주기초`(교대 공급) 명령은 **제거**되었고, 독립 OFF용 `roff`/`toff`와 전체 정지 `stop`이 추가되었습니다.

### 위즈 탱크

| 항목 | 사양 |
|------|------|
| 용기 | 다이소 락앤락 김치통 (5L) |
| 내용물 | 참조 해수 (알려진 dKH 값) |
| 센서 | DS18B20 온도 센서 침수 |

모든 비이커는 위즈 탱크 안에 담겨 있어 참조수와 동일한 온도 환경을 유지합니다.

> ## ★밀폐 공통 헤드스페이스 — 선택이 아니라 필수
>
> **참조수 통·측정 챔버·펌프 흡입을 *하나의 공통* 밀폐 공기로 묶습니다(폐루프 흡입).** 차동식 `KH_tank = KH_ref × 10^(−ΔpH)`이 순수 KH차를 주려면 측정 순간 ref·tank가 **같은 pCO₂**여야 하는데, 밀폐 공통 헤드스페이스는 두 물이 *같은 한 덩어리 공기*를 공유하게 해 이 조건을 **본질적으로 보장**합니다.
>
> 1. **방 CO₂ 변동 차단(핵심):** 외부 공기를 끊으므로 방 CO₂가 ΔpH로 새어들 통로가 없어집니다. 차동 상쇄가 가장 잘 일어나는 조건.
> 2. **증발 차단:** 측정수의 증발 농축(경도 상승)을 막아 ΔKH=0 전제를 지키고 참조수 dKH도 장기간 일정하게 유지합니다.
> 3. **온도 균일:** 밀폐 공간 내 참조수·수조수·KCL이 동일 온도로 유지되어 Nernst 보정 정확도가 향상됩니다.
>
> **★필수 조건 — 분리 밀폐 금지.** ref와 tank가 **서로 다른** 밀폐 공간에 있으면 두 헤드스페이스가 따로 드리프트해 상쇄가 깨집니다. 반드시 **하나의 공통 헤드스페이스**를 공유해야 합니다. 펌프 흡입도 외기가 아니라 이 공통 공기에서 폐루프로 끌어옵니다.
>
> **검증(2026-06-19):** 밀폐 적용 후 첫 새벽(고-CO₂) 측정에서 그간 3회 재현되던 새벽 오차(ΔpH −0.043~−0.059)가 **−0.008**로 붕괴했습니다(낮 baseline 수준). 또 헤드스페이스가 참조수 알칼리니티 버퍼로 핀고정돼 평형이 사실상 즉시 도달(과거 새벽 33회 → 4회, 측정 ~50분 → ~5분) → tank↔ref 측정 시간차가 붕괴해 드리프트성 오차까지 구조적으로 사라졌습니다. 무보정 원시오차도 −0.78 → −0.165 dKH로 줄어 목표 정확도 안에 들어왔습니다. 상세는 [측정 대장](measurement-ledger.md).
>
> **주의:** 펌프 발열이 헤드스페이스/수온을 밀어올릴 수 있으므로 온도욕조 안정과 응결을 점검하세요(6/19 측정은 28.1℃로 폭주 없음).

## 자동 측정 시퀀스 (PC 측 V4)

### 설계 배경 — 폭기 중 평형 추종

차동식 `KH_tank = KH_ref × 10^(−ΔpH)`이 순수 KH차를 반영하려면 측정 순간
ref·tank가 **같은 pCO₂**여야 합니다. 이를 두 축으로 달성합니다.

1. **밀폐 공통 헤드스페이스(필수, 가스 기준 고정):** ref·tank가 하나의 밀폐 공기를
   공유하므로 *같은 pCO₂*가 본질적으로 보장되고 방 CO₂ 변동 통로가 차단됩니다 →
   [위즈 탱크 — 밀폐 공통 헤드스페이스](#위즈-탱크) 참조.
2. **폭기 중 평형 추종:** V4는 **폭기를 유지한 채** tank·ref가 각각 **진짜
   평형(평탄)에 도달할 때까지** 반복 측정합니다 — 폭기를 끄고 닫아두는 방식이
   아니라 두 물을 같은 평형(천장)으로 끌고 갑니다.

밀폐가 평형의 기준(헤드스페이스 pCO₂)을 고정하므로 평형 추종은 *움직이는* 천장이
아니라 *고정된* 천장을 향하게 되어, 측정 시간차(gap)에 따른 드리프트 오차까지
사라집니다. (근거·실측은 [측정 대장](measurement-ledger.md) 참조: 6/18 개방 셋업의
ΔpH −0.042/−0.043가 방 CO₂ 스윙과 무관하게 일정함을 보였고, **6/19 밀폐 적용 후
새벽 ΔpH가 −0.008로 붕괴**해 새벽 오차가 제거됨.)

- **폭기 중 측정 + 평탄 판정**: `tank`/`ref`를 30초 간격으로 반복하며 **최근 4개
  측정의 정수 milli-pH (max−min) ≤ 2 mpH**면 평형으로 보고 종료(윈도우 span 판정).
  정수 비교라 float 지터가 없고, 느린 크리프에 조기 종료되지 않습니다.
- **2-phase**: **[A]** tank·ref 동시 폭기 + tank 평탄까지(ref는 홀딩 챔버에서
  co-aeration) → 전이(무대기) → **[B]** ref 평탄까지(co-aeration이라 평형 근처서
  시작 → 빨리 수렴).
- **무한 대기 방지**: phase별 최대 측정시간 `PHASE_MAX_SECS=7200`s(=2h), 최대 횟수
  `MEAS_MAX=240`, 연속 측정실패 `FAIL_MAX=5`, 시리얼 read 타임아웃. 미평탄 상한
  도달 시 마지막값 채택 + 경고(`dkh.dat`에 음수 표식).
- **트림평균 전위**: 펌웨어가 64샘플을 정렬해 상·하위 16개를 버리고 중앙 32개만
  평균 → 폭기 버블 스파이크 제거(차동이라 절대 pH만 미세 변동, dKH 무영향).
- **프로브 보관액 3M KCl**: 측정 외 상시 소크가 곧 컨디셔닝이라 별도 재생 루틴
  불필요. 중성염이라 잔류(캐리오버)가 dKH에 무영향 → V4는 헹굼 단계 생략.

### 전체 흐름 (measure_kh_once.py)

배관(2026-06-20): `m1`=본수조↔홀딩, `m2`=홀딩↔측정챔버, `m3`=KCl↔측정챔버,
`m4`=참조수(5L 위즈수조)↔측정챔버. **측정 순서 = tank 먼저 → ref 나중** —
tank 측정 내내 5L 위즈수조를 *동시 폭기*해 ref 가 5L서 co-aeration → ref 차례엔
이미 평형 근처라 빨리 수렴합니다. (헹굼 단계는 V4에서 제거됨.)

```
── 준비 ──────────────────────────────────────────
  airoff → ton
  ① m3b:68  KCl 보관액 배출 (측정 챔버 비우기)
  ② m1f:70  본수조수 → 홀딩 챔버
  ③ m2f:60  홀딩 → 측정 챔버 (tank 측정수 채움)

── [A] tank 평탄까지 ─────────────────────────────
  airoff → ron   (측정 챔버 tank + 5L 위즈수조 *동시* 폭기)
  ④ tank 평탄 측정 (30초 간격 반복, span4≤2mpH AND net8≤1mpH)
       ※ 이 동안 5L 위즈수조가 co-aeration → ref 미리 평형 근처로

── 전이 (무대기: 홀딩 파킹으로 ref 측정 조기 시작) ──
  airoff → ton
  ⑤ m2b:68  측정 챔버 tank → 홀딩 *임시 파킹* (완전 배출 대신 빠른 비움)
  ⑥ m4f:60  참조수 5L → 측정 챔버 (동시폭기로 이미 평형 근처)

── [B] ref 평탄까지 ──────────────────────────────
  airoff → ron   (측정 챔버 ref + 5L 위즈수조 *동시* 폭기)
  ⑦ ref 평탄 측정 (co-aeration이라 빨리 수렴) → airoff
  ⑧ calkh    dKH 계산 + 출력 + 이력 저장
       └ calref 모드(--setref): calkh 대신 calref = ref dKH 역산·EEPROM 저장

── 정리 ─────────────────────────────────────────
  ton
  ⑨ m4b:70  참조수 측정 챔버 → 5L 위즈수조 회수 (긴 호스 +10)
  ⑩ m1b:82  파킹된 tank 홀딩 → 본수조 마무리 배출
  ⑪ m3f:60  KCl 재공급 (프로브 소크) → airoff
```

> 펌웨어는 측정 순서와 무관합니다. `ref`/`tank` 플래그가 독립 저장되고
> `calkh`/`calref`는 둘 다 갖춰진 뒤 계산하므로 tank 우선이어도 동작이 같습니다.
>
> **★규칙:** 모든 액체 이동(`mXf`/`mXb`) 직전에 `airoff`로 폭기를 끄고, 펌프 구동 전
> `ton`으로 PWM 제어기를 켭니다. 폭기(`ron`)는 *액체 정지 구간*(측정)에만.
>
> **★종료 상태:** V4는 측정 후 `airoff`(KCl 소크 상태)로 끝납니다 — 구 버전의
> "측정 사이 기포기 ON 유지(terminal ron)"는 **제거**되었습니다.
>
> 오류/비정상 종료 시 `_safe_cleanup`(에어 OFF → 측정 챔버 배출 `m2b:68`→`m1b:82`
> → KCl 소크 `m3f:60` 복원)이 돌고, KCl 소크 미검증이면 에러로 `0.0`을 기록합니다.

| `bin/measure_kh_once.py` | **V4** — 1회 측정 후 종료 (Windows 작업 스케줄러 정시 실행용) |
| `bin/test_aeration_plateau.py` | tank 단일 평형곡선 진단 (V4와 동일 평탄 기준, 종료 시 KCl 소크) |
| `bin/firmware_sim.py` | 소켓 가상 포트 펌웨어 시뮬레이터 (회귀 테스트용, WSL 전용) |

| `bin/bt_health.py` | (보조) HC-06 BT 링크 드롭률·RTT 모니터 |

**V4 (`measure_kh_once.py`) 주요 상수:**

| 상수 | 기본값 | 의미 |
|------|--------|------|
| `FLAT_SPAN_N` | 4 | 흔들림(span) 판정 윈도우 — 최근 N개 |
| `FLAT_SPAN_MPH` | 2 | 최근 `FLAT_SPAN_N`개 정수 milli-pH의 (max−min) ≤ 2면 흔들림 통과 |
| `FLAT_NET_N` | 8 | 드리프트(net) 판정 룩백 — span보다 긴 창(★B1, 느린 단조 꼬리 조기 latch 차단) |
| `FLAT_NET_MPH` | 1 | 최근 `FLAT_NET_N`개 양끝차 ≤ 1이면 드리프트 통과(span·net 둘 다 통과해야 평형) |
| `MEAS_INTERVAL` | 30 | 측정 간 간격(초) — 폭기 지속 |
| `PHASE_MAX_SECS` | 7200 | phase(tank/ref)별 최대 측정시간(초)=2h, 초과 시 마지막값+경고 |
| `MEAS_MAX` | 240 | phase별 최대 측정 횟수(백스톱)=7200/30 |
| `FAIL_MAX` | 5 | 연속 측정 파싱 실패 허용 → 초과 시 phase 실패 |
| `SEND_RETRY_MAX` | 3 | RF 순단 시 명령 재시도 횟수 |
| `KEEPALIVE_SECS` | 12 | 유휴 keepalive 간격(초) — HC-06 ~20초 유휴 드롭 예방 |
| `MEAS_READ_TIMEOUT` | 20 | 측정 1회 `[OK]` 대기 상한(초) |
| `LINK_PING_TIMEOUT` | 3 | `ensure_link`/`reconnect` 핑 응답 대기 상한(초) |

> RF 순단(블루투스) 대응 상수(`SEND_RETRY_MAX`·`KEEPALIVE_SECS`·`RECONNECT_TRIES` 등)와 그 동작은 [블루투스 통신 안정화 & 시뮬레이터 검증](bt-reconnect-and-testing.md)에서 자세히 다룹니다.

> 폭기 시간은 **고정값이 아니라 평형(평탄)까지 적응**합니다(구 `AIR_SECS`/`STABLE`/`CONV_EPS` 폐기). 측정 펌프초: tank 채움(본수조→홀딩) `m1f:70`·완전 배출(홀딩→본수조) `m1b:82`, 이송(홀딩↔측정챔버) `m2f:60`/`m2b:68`, 참조수(5L↔측정챔버) `m4f:60`/`m4b:70`(긴 호스라 역방향 +10), KCl `m3b:68`/`m3f:60`.

**`dkh.dat` 규약** (`HH ref_pH tank_pH ref_kh(8.448) tank_kh temp`):

- **에러 표식 = 5값 전부 0** (측정 실패/타임아웃/KCl 소크 실패). **에러 래치**: 마지막 줄이 에러 표식이면 다음 실행은 측정 없이 에러 표식만 재기록 — 수동으로 마지막 에러 줄을 지우기 전까지(오류 무인 반복 방지).
- **평탄 미도달 = `tank_kh` 음수(−)** 표식 (값 크기는 유효, 서버가 부호로 인지). 전부-0 에러 표식과 구분되어 래치 안 걸림.
- 평탄 도달 여부·측정 횟수는 로그(`measure_kh.log`)와 결과 헤더(`평탄도달 : tank N회 O / ref N회 O`)에 남습니다.

### 1회 측정 실행법 (`measure_kh_once.py`)

1회만 측정하고 종료하는 스크립트입니다. Windows 작업 스케줄러가 정시(05:00/13:00/21:00)에 호출하지만, 수동 실행도 가능합니다.

**동작:** 준비 → **[A] tank·ref 동시 폭기 + tank 평탄까지** → 전이 → **[B] ref 평탄까지** → `calkh` → 정리 후 `airoff`(KCl 소크)로 종료. 평형까지 적응하므로 1회 소요는 가변이며(방 CO₂·시작 격차 의존) **대략 35~50분**, phase 상한(각 7200s=2h, 두 phase 합 최대 4h = 측정 갭 8h의 절반)에 걸리면 더 길어질 수 있습니다.

**포트 지정:** 첫 번째 명령행 인자로 포트를 넘길 수 있고, 없으면 스크립트 내 `PORT` 상수(현재 `COM9`)를 씁니다.

**모드:** 인자가 없으면 `calkh`(수조 dKH 측정 → `dkh.dat` 기록). `--setref <수조실측dKH>` 를 주면 `calref`(참조수 dKH 역산·EEPROM 저장, `dkh.dat` 기록 안 함) — 자세한 내용은 [사용설명서 §3.2](user-manual.md#32-자동-측정-pc-연동)를 참조하세요.

```bash

# 포트 직접 지정

# ref dKH 교정(역산) — 알려진 수조 dKH 기준
python measure_kh_once.py --setref 8.448

# 콘솔 없이 실행 (스케줄러 방식) — 출력은 C:\dkh\measure_kh.log 로 redirect
pythonw measure_kh_once.py

# WSL에서 Windows 파이썬으로 수동 실행 (cp949 인코딩 오류 회피용 -X utf8 필수)
/mnt/c/dkh/python313/python.exe -X utf8 'C:\dkh\work\measure_kh_once.py' 2>&1
```

**PowerShell에서 직접 실행** (정션 `C:\dkh` 경로 사용, `-X utf8` 필수):

```powershell

# 포트 직접 지정

# ref dKH 교정 (calref) — 수조 실측 8.448 dKH 기준으로 참조수 dKH 역산·저장
```

### Windows 작업 스케줄러 등록 (정시 자동 측정)

`measure_kh_once.py`를 **매일 05:00 / 13:00 / 21:00**(05시부터 8시간 간격, 하루 3회) 자동 실행하는 예입니다. 한 개의 반복 패턴(Repetition) 대신 **고정 시각 Daily 트리거 3개**를 씁니다 — 매일 재무장되어 재부팅 후에도 스케줄이 깨지지 않고, 꺼져 있던 시간대의 측정은 보충하지 않습니다(의도된 동작).

**관리자 PowerShell**에서 실행합니다(한글 경로 문제를 피하기 위해 ASCII 정션 `C:\dkh` 경로 사용):

```powershell
# 1) 하루 3회 고정 시각 트리거 (05시 + 8h 간격)
$triggers = @(
    New-ScheduledTaskTrigger -Daily -At 05:00
    New-ScheduledTaskTrigger -Daily -At 13:00
    New-ScheduledTaskTrigger -Daily -At 21:00
)

# 2) 콘솔 없는 pythonw 로 1회 측정 스크립트 실행
$action = New-ScheduledTaskAction `
    -Execute 'C:\dkh\python313\pythonw.exe' `
    -Argument 'C:\dkh\work\measure_kh_once.py'

# 3) 설정: 배터리·유휴로 중지 금지, 1회 측정 상한 270분(스크립트 자체 최대 ~4h + 여유), 중복 실행 시 새 인스턴스 무시
$settings = New-ScheduledTaskSettingsSet `
    -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries `
    -ExecutionTimeLimit (New-TimeSpan -Minutes 270) `
    -MultipleInstances IgnoreNew
$settings.IdleSettings.StopOnIdleEnd = $false

# 4) 로그인한 대화형 세션의 사용자 토큰으로 실행 (COM 포트는 사용자 세션에 바인딩됨)
$principal = New-ScheduledTaskPrincipal -UserId $env:USERNAME -LogonType Interactive

Register-ScheduledTask -TaskName 'Measure KH' -TaskPath '\tsyi\' `
    -Trigger $triggers -Action $action -Settings $settings -Principal $principal -Force
```

설정값의 이유:

| 설정 | 값 | 이유 |
|------|----|------|
| `AllowStartIfOnBatteries` / `DontStopIfGoingOnBatteries` | 켬 | 노트북에서 기본값(배터리 시 중지)이 데몬·측정을 로그 없이 강제 종료하는 문제 회피 |
| `IdleSettings.StopOnIdleEnd` | `$false` | 유휴 종료 시 작업이 중단되지 않도록 |
| `ExecutionTimeLimit` | 270분 | V4 1회 측정은 평형까지 적응하므로 대개 **35~50분**이지만, tank·ref 두 phase가 모두 상한(각 `PHASE_MAX_SECS`=2h)에 걸리면 측정만 최대 4h + 이송·정리 여유가 필요. 스크립트 자체 phase 상한이 런타임을 4h로 묶으므로, 정당한 장시간 측정을 죽이지 않도록 그보다 넉넉한 270분으로 둠(멈춘 프로세스는 시리얼 타임아웃·`FAIL_MAX`로 스크립트가 자체 종료) |
| `MultipleInstances` | `IgnoreNew` | 직전 측정이 진행 중이면 다음 정시 트리거는 스킵 |
| `LogonType` | `Interactive` | COM(블루투스) 포트가 사용자 세션에 바인딩되어, 로그인된 대화형 세션에서만 접근 가능 |

> **로그인 세션 필요:** 로그아웃하면 세션·토큰이 파괴되어 작업이 죽습니다. 자리를 비울 땐 로그아웃 대신 **잠금(Win+L)** 을 쓰면 세션이 유지되어 계속 동작합니다(절전·디스플레이 OFF는 무방). 자동 복귀가 필요하면 자동 로그인을 권장합니다.
>
> 복잡한 PowerShell은 bat에 인라인으로 쓰지 말고 `.ps1`로 분리한 뒤 `register_measure_kh.bat`에서 `powershell -ExecutionPolicy Bypass -File "%~dp0register_measure_kh.ps1"` 한 줄로 호출합니다(한글 경로 + `^` 줄연속 조합 인코딩 오류 회피).

### 블루투스 통신 안정화 & 시뮬레이터 검증

측정은 HC-06 블루투스 SPP(COM9)로 통신하는데, 이 RF 링크가 간헐적으로 끊긴다(주로 **명령을 보내려는 순간 이미 끊겨 있음**). 이에 대응해 `measure_kh_once.py`의 `send()`는 다음을 보장한다.

1. **송신 전 연결 확인**(`ensure_link` = `status` 핑) — 모든 명령은 보내기 전 링크 생존을 확인
2. 끊겼으면 **재연결**(`reconnect` = 포트 `close→open` + 펌웨어 응답 확인)
3. 보낸 뒤 연결 문제(예외·응답 미수신)면 **재연결 후 재시도**(`SEND_RETRY_MAX`회)
4. **모터는 재시도 시 먼저 정지(`mNs`) 후 재송신** — 펌프 중복 구동 방지
5. (예방) 유휴 구간 **keepalive** — 12초마다 빈 줄로 ~20초 유휴 드롭 차단

호스트측으로는 Wi-Fi/BT 네트워크 장치 비활성화·USB 선택적 절전 OFF로 2.4GHz 간섭·포트 절전을 줄인다.

**회귀 검증(배포 전 필수):** 소켓 가상 펌웨어 시뮬레이터로 정상·예외 상황을 검증한다.

```bash
cd bin && python3 test_measure_sim.py     # WSL, 12 시나리오/58 검증 — 전부 PASS여야 배포
```

> **[블루투스 통신 안정화 (RF 순단 대응) 및 시뮬레이터 회귀 검증](bt-reconnect-and-testing.md)** 에 상세히 정리되어 있다.

## 온도 환경

위즈 탱크(5L) 안에 모든 비이커(200ml)가 담겨 있으므로:

- 참조수, 수조수 샘플, 3M KCl이 모두 동일 온도로 유지됨
- DS18B20이 위즈 탱크 수온을 측정하여 Nernst 온도 보상에 사용

## 측정 정확도

### Nernst 온도 보정

pH 전극의 기울기는 온도에 따라 변합니다 (Nernst 방정식: slope = 0.19842 × T(K) mV/pH). 보정 온도를 EEPROM에 저장하여 보정 시와 측정 시 온도가 달라도 정확하게 보상합니다.

```
pH_corrected = 7.0 + (pH_raw − 7.0) × (273.15 + T_cal) / (273.15 + T_meas)
```

보정 온도 저장 후 **15°C~35°C 전 범위에서 Nernst 보정 오차 0%**입니다.

### 오차 요인 분석 (dKH 8.5 기준)

| 요인 | pH 오차 | dKH 오차 | 비고 |
|------|--------|---------|------|
| ADS1115 ±1 LSB | ±0.0007 pH | ±0.010 | 16비트 ADC |
| 64회 오버샘플링 | ÷8 | ±0.001 | √64 = 8배 개선 |
| Nernst 보정 | 0 | 0 | 보정 온도 EEPROM 저장 |
| DS18B20 ±0.5°C | ±0.0017 pH | ±0.023 | 온도 센서 정밀도 |
| **pH 프로브 반복성** | **±0.005 pH** | **±0.070** | **지배적 오차 요인** |
| **전체 (RSS)** | | **±0.073 dKH (±1.2%)** | |

### 시스템 성능 요약

```
측정 분해능:    ±0.001 dKH (소수점 3자리)
시스템 정확도:  ±0.07 dKH (±1.2%)
유효 자릿수:    소수점 이하 2자리
```

> 정확도의 97%는 pH 프로브(유리전극) 자체의 반복성에 의해 결정됩니다. ADC, Nernst 보정, 펌웨어 계산 로직은 프로브 성능을 충분히 살릴 수 있는 수준입니다.

### 실측 안정성 (KCL 용액, 5회 연속)

```
전압 편차: ±0.061 mV
pH 편차:   ±0.0005 pH
온도:      28.6°C 일정
```

> ADS1115 16비트 분해능의 이론적 한계에 근접하는 안정성입니다.

### 진단: 프로브 vs 셋업 분리 (널 테스트)

측정 시 참조수 pH와 수조수 pH 사이에 ΔpH 오프셋이 남을 수 있습니다(두 물이 동일 해수라면 이상적으로 ΔpH = 0). 이 오프셋의 원인이 **프로브·경로의 측정 아티팩트**인지, **두 물의 실제 화학 차이(CO2 탈기 비대칭)**인지 가르는 절차입니다.

**왜 재보정이 답이 아닌가:** dKH는 차동식(`KH_tank = KH_ref × 10^(−ΔpH)`)이라, pH 보정의 offset 오차는 참조·수조 양쪽에 똑같이 실려 ΔpH에서 상쇄되고, slope 오차도 작은 ΔpH(보통 0.02~0.07)에 비례하므로 영향이 미미합니다. 따라서 오프셋을 재보정(`calph`)으로 잡으려 하면 노이즈만 늘 뿐이며, 먼저 원인을 분리해야 합니다. (재보정은 절대 pH 동작점을 ADC 선형구간 안으로 유지하는 **예방·점검**용입니다.)

**절차:** **동일 해수**를 참조 경로와 수조 경로 양쪽에 넣고(진짜 ΔKH=0) V4로 측정합니다. V4는 양쪽을 폭기 중 진짜 평형까지 끌고 가므로, 남는 ΔpH가 프로브·경로 아티팩트인지 판별하기 좋습니다.

| 결과 | 해석 | 조치 |
|------|------|------|
| ΔpH ≈ 0 | 프로브·경로 정상, 차동이 방 CO₂를 잘 상쇄 | 그대로 운용 |
| ΔpH = 작은 상수 (방 CO₂ 무관, 재현) | 구조 baseline 후보 (경로 비대칭 등) | **영점보정은 독립 재현 확인 후** — 동일 해수 새 널테스트로 일정 vs 우연 판정 ([측정 대장](measurement-ledger.md)) |
| ΔpH가 방 CO₂ 따라 변동 | 폭기/평형 비대칭(미완) 또는 외기 유입 | **밀폐 공통 헤드스페이스 확인**(분리 밀폐·누설 점검), 평탄 도달 확인(`평탄도달 O`) |

> **현황(2026-06):** 개방 셋업(200mL 개방 챔버 tank + 5L 반밀폐 ref) 널테스트에서 V4가 방 CO₂ 의존성을 제거(ΔpH −0.042/−0.043 동일)했으나 baseline은 상수가 아니었고(13:00 −0.026, +0.29 dKH = 증발 confound 의심), 무보정 원시오차도 −0.78로 컸습니다. **2026-06-19 밀폐 공통 헤드스페이스 적용 후 첫 새벽 측정에서 ΔpH가 −0.008로 붕괴**(새벽 오차 제거)하고 원시오차도 −0.165로 줄어 목표 정확도 안에 들어왔습니다 — 밀폐가 새벽 CO₂와 증발 두 잔여 오차원을 동시에 덮은 것으로 보입니다. 다만 단일 점이므로 13:00 밀폐 측정(증발 흩어짐 판정)·시계열 누적까지는 **영점보정 확정을 보류**합니다. 상세는 [측정 대장](measurement-ledger.md).

**프로브 건강 계기:** 별도 장비 없이 `measure_kh.log`의 **평탄 도달까지 측정 횟수**(`tank N회 / ref N회`)로 전극·평형 상태를 모니터링합니다. 같은 조건에서 횟수가 점차 늘면 응답 둔화(노화) 또는 접합부(KCl junction) 오염 신호이며, 이때 묽은 HCl 접합 세척을 검토합니다. 접합전위 1 mV 변동 ≈ 0.017 pH ≈ 0.3 dKH이므로 무시할 수 없습니다. (단, 평탄 횟수는 폭기 평형시간·시작 격차에도 좌우되므로 추세로 해석.)
