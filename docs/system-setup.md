## 구성 요소

### 제어기 본체

| 구성 | 사양 |
|------|------|
| MCU | Arduino Nano V3.0 (ATmega328P) |
| pH 센서 | DFRobot SEN0161-V2 + ADS1115 16bit ADC |
| 온도 센서 | DS18B20 (위즈 탱크 내 침수) |
| 통신 | HC-06 블루투스 (9600 baud) |
| 모터 드라이버 | L298N1/L298N2 (도징 펌프, 12V) + L298N3 (에어 펌프, USB 5V) |
| 속도 제어 | PWM 컨트롤러 Motor− 출력 → L298N ENA/ENB (역 PWM) |
| 전원 | 12V DC → Buck 5V (MCU) + USB 5V (L298N3 에어 펌프) |

### 연동 펌프 (4구, 모두 양방향)

| 펌프 | 경로 | 정방향 (f) | 역방향 (b) |
|------|------|------------|------------|
| 펌프1 (M1) | 수조 ↔ 수조물 비이커 | 수조 → 수조물 비이커 (샘플링) | 수조물 비이커 → 수조 (반환) |
| 펌프2 (M2) | 수조물 비이커 ↔ pH 측정 비이커 | 수조물 → pH 비이커 (측정 이송) | pH 비이커 → 수조물 (측정 후 반환) |
| 펌프3 (M3) | 3M KCl 비이커 ↔ pH 측정 비이커 | KCL → pH 비이커 (프로브 보관) | pH 비이커 → KCL 비이커 (저장수 배출) |
| 펌프4 (M4) | 위즈 탱크 ↔ pH 측정 비이커 | 참조수 → pH 비이커 (측정 이송) | pH 비이커 → 위즈 탱크 (참조수 반환) |

### 에어 공급 및 PWM 속도 제어 (L298N3)

| 핀 | 명령 | 용도 |
|-----|------|------|
| D12 (L298N3 채널A) | `ron` | USB 5V 에어 펌프 **양쪽 동시 ON** (참조수 + 수조수 폭기) |
| D13 (L298N3 채널B) | `ton` | PWM 속도 제어기 ON (모터 구동 시 활성화) |
| D12 + D13 | `airoff` | 양쪽 동시 LOW (에어 OFF + PWM 제어기 OFF) |

- `ron`: D12 HIGH → 에어 펌프 2개 동시 ON (참조수·수조수 동시 탈기)
- `ton`: D13 HIGH → PWM 속도 제어기 활성화 (도징 펌프 구동 전 ON)
- `airoff`: D12·D13 동시 LOW → 에어 정지 + PWM 제어기 비활성화

> **참고:** 기존 `air:총초:주기초` 교대 공급 명령은 deprecated되었습니다.

### 위즈 탱크

| 항목 | 사양 |
|------|------|
| 용기 | 다이소 락앤락 김치통 (5L) |
| 내용물 | 참조 해수 (알려진 dKH 값) |
| 센서 | DS18B20 온도 센서 침수 |
| 비이커 | 200ml x 3개 (수조물, pH 측정, 3M KCl) |

모든 비이커는 위즈 탱크 안에 담겨 있어 참조수와 동일한 온도 환경을 유지합니다.

> **밀폐가 필수인 이유:**
> - **참조수 보관:** 증발에 의한 농도 변화와 외부 오염을 차단하여 dKH를 장기간 일정하게 유지
> - **측정 시:** 폭기(CO₂ 평형) 완료 후 뚜껑을 닫아 대기 CO₂ 재흡수를 차단해야 pH가 안정 유지됨. 개방 상태에서는 CO₂가 재흡수되어 pH가 지속 변동하므로 정확한 측정 불가
> - **온도 균일:** 밀폐 공간 내 참조수·수조수·KCL이 동일 온도로 유지되어 Nernst 보정 정확도 향상

## 자동 측정 시퀀스 (대칭화 설계, 2026-06)

### 설계 배경

구 시퀀스는 참조수가 pH 비이커에서 30분 폭기되는 동안 프로브가 참조수에
완전 평형(30분 소크)된 후 측정되는 반면, 수조수는 교체 후 약 20초 만에
측정되어 **전극 응답 지연이 수조수 측정에만 실리는 계통 편향**이 있었습니다.
대칭화 설계는 이 지연을 양쪽에 동일하게 실어 공통모드로 상쇄합니다.

- **저장수조(위즈 탱크 5L) 직접 폭기**: 참조측 에어스톤을 pH 비이커에서
  위즈 탱크 안으로 이전. 5L 열·화학 관성으로 참조수 평형이 안정되고,
  폭기 중 프로브는 pH 비이커의 KCl에 소크 상태로 휴식
- **tank 우선 측정**: KCl 스윕(짧은 왕복)이 tank 측정 전 헹굼을 겸해 단계 절약
- **타이밍 대칭**: tank/ref 블록의 헹굼·채움·안정화 시간을 초 단위로 동일하게
- **수렴 판정**: `tank`/`ref` 측정을 45초 간격으로 반복, 연속 차 < 0.003 pH면
  진행(최대 6회). 수렴 횟수 증가 = 전극 응답 둔화 신호(건강 지표)
- **프로브 보관액 3M KCl**: 측정 외 상시 소크가 곧 컨디셔닝이라 별도 재생
  루틴 불필요. 중성염이라 스윕 혼입이 dKH에 무영향

### 전체 흐름

```
── 준비 ──────────────────────────────────────────

① 수조수 샘플링
   펌프1 정방향: 수조 → 수조물 비이커 (m1f)
   → 프로브는 pH 비이커의 3M KCl에 소크 유지

── 폭기 (CO₂ 평형, 20분) ────────────────────────

② ron: 위즈 탱크(참조수 5L) + 수조물 비이커 동시 폭기
   완료 후 airoff, 위즈 탱크 뚜껑을 닫아 대기 CO₂ 재흡수 차단

── tank ─────────────────────────────────────────

③ KCl 보관액 배출
   펌프3 역방향: pH 비이커 → KCl 비이커 (m3b)

④ KCl 스윕 = 헹굼
   펌프2: m2f:2 → m2b:2 (쓸린 KCl 잔량은 수조물 비이커로)

⑤ 수조수 채움 → 측정 → 배출
   m2f → 60초 안정화 → tank 수렴 측정 → m2b

── ref (tank 블록과 헹굼·채움·안정화 타이밍 동일) ──

⑥ 참조수 헹굼
   펌프4: m4f:2 → m4b:2 (tank 잔막 제거,
   반환수는 KCl-free → 5L 참조수 앵커 보호)

⑦ 참조수 채움 → 측정 → 계산 → 배출
   m4f → 60초 안정화 → ref 수렴 측정 → calkh → m4b

── 정리 ─────────────────────────────────────────

⑧ 챔버수 방출
   펌프1 역방향: 수조물 비이커(KCl 미량 포함) → 수조 (m1b, 무한 희석)

⑨ KCl 재공급 (프로브 보관)
   펌프3 정방향: KCl 비이커 → pH 비이커 (m3f)

⑩ ron: 측정 사이 기포기 ON 유지
   다음 측정까지 밀폐 참조수를 지속 탈기 (ΔpH 폭기부족 오프셋 대응)
```

> 펌웨어는 측정 순서와 무관합니다. `ref`/`tank` 플래그가 독립 저장되고
> `calkh`는 둘 다 갖춰진 뒤 계산하므로 tank 우선이어도 동작이 같습니다.
>
> 위 흐름의 펌프 동작 초(`m2f:2` 등)는 예시이며, 실제 정시 측정(`measure_kh_once.py`)은 헹굼·채움 60초 / 배출 68초, 수조수 호스가 긴 m1은 80 / 92초를 씁니다.

### 구현: PC측 스크립트 (bin/)

수렴 판정(측정값 비교 후 반복 여부 결정)이 필요하므로 펌웨어 `seq` 명령이
아닌 PC측 파이썬 스크립트가 명령을 단계별로 전송합니다.

| 스크립트 | 동작 |
|----------|------|
| `bin/measure_kh_once.py` | 1회 측정 후 종료 (Windows 작업 스케줄러 정시 실행용) |
| `bin/measure_kh_loop.py` | 매 정시(HH:00) 무한 반복 |

| 상수 | 기본값 | 의미 |
|------|--------|------|
| `AIR_SECS` | 1800 | 폭기 시간(초) |
| `STABLE_SECS` | 60 | 채움 후 안정화(초) — tank/ref 동일 |
| `CONV_INTERVAL` | 45 | 수렴 재측정 간격(초) |
| `CONV_EPS` | 0.003 | 수렴 기준 (연속 측정 pH 차) |
| `CONV_MAX` | 6 | 최대 측정 횟수 |

결과는 `dkh.dat`에 `HH ref_pH tank_pH ref_kh tank_kh temp` 형식으로 기록되고
(오류 시 0.0), 수렴 횟수는 로그(`measure_kh.log`)에 남습니다.

> 위 기본값은 `measure_kh_loop.py` 기준입니다. `measure_kh_once.py`는 `AIR_SECS=1200`, `CONV_EPS=0.005`로 다릅니다.

### 1회 측정 실행법 (`measure_kh_once.py`)

1회만 측정하고 종료하는 스크립트입니다. Windows 작업 스케줄러가 정시(05:00/13:00/21:00)에 호출하지만, 수동 실행도 가능합니다.

**동작:** 폭기(20분) → tank 측정 → ref 측정 → `calkh` → 정리 후, 마지막에 `ron`으로 기포기를 켠 채 종료해 다음 측정까지 참조수를 지속 탈기합니다. 1회 소요 약 37~44분.

**포트 지정:** 첫 번째 명령행 인자로 포트를 넘길 수 있고, 없으면 스크립트 내 `PORT` 상수(현재 `COM15`)를 씁니다.

```bash
# 콘솔에서 출력을 보며 실행 (포트 = 스크립트 기본값 COM15)
python measure_kh_once.py

# 포트 직접 지정
python measure_kh_once.py COM7

# 콘솔 없이 실행 (스케줄러 방식) — 출력은 C:\dkh\measure_kh.log 로 redirect
pythonw measure_kh_once.py

# WSL에서 Windows 파이썬으로 수동 실행 (cp949 인코딩 오류 회피용 -X utf8 필수)
/mnt/c/dkh/python313/python.exe -X utf8 'C:\dkh\work\measure_kh_once.py' 2>&1
```

> - **포트(COM15)는 블루투스 페어링에 묶입니다.** 평소 유지되지만 장치 삭제 후 재페어링하면 번호가 바뀌므로, 그때 실제 포트를 인자로 넘기거나 `PORT` 상수를 수정합니다.
> - 측정 포트는 스케줄러 작업과 공유하므로, **정시 측정(05/13/21시)이 진행 중일 때 수동 실행하면 시리얼 충돌**이 납니다.
> - `pythonw`는 `sys.stdout`이 `None`이라, 스크립트가 모든 출력을 `C:\dkh\measure_kh.log`로 보냅니다(1MB 초과 시 새로 시작).

### Windows 작업 스케줄러 등록 (정시 자동 측정)

`measure_kh_once.py`를 **매일 05:00 / 13:00 / 21:00**(05시부터 8시간 간격, 하루 3회) 자동 실행하는 예입니다. 한 개의 반복 패턴(Repetition) 대신 **고정 시각 Daily 트리거 3개**를 씁니다 — 매일 재무장되어 재부팅 후에도 스케줄이 깨지지 않고, 꺼져 있던 시간대의 측정은 보충하지 않습니다(의도된 동작).

**관리자 PowerShell**에서 실행합니다(한글 경로 문제를 피하기 위해 ASCII 정션 `C:\dkh` 경로 사용):

```powershell
# 1) 하루 3회 고정 시각 트리거 (05시 + 8h 간격)
$triggers = @(
    New-ScheduledTaskTrigger -Daily -At 05:00
    New-ScheduledTaskTrigger -Daily -At 13:00
    New-ScheduledTaskTrigger -Daily -At 21:00
)

# 2) 콘솔 없는 pythonw 로 1회 측정 스크립트 실행
$action = New-ScheduledTaskAction `
    -Execute 'C:\dkh\python313\pythonw.exe' `
    -Argument 'C:\dkh\work\measure_kh_once.py'

# 3) 설정: 배터리·유휴로 중지 금지, 1회 측정 상한 55분, 중복 실행 시 새 인스턴스 무시
$settings = New-ScheduledTaskSettingsSet `
    -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries `
    -ExecutionTimeLimit (New-TimeSpan -Minutes 55) `
    -MultipleInstances IgnoreNew
$settings.IdleSettings.StopOnIdleEnd = $false

# 4) 로그인한 대화형 세션의 사용자 토큰으로 실행 (COM 포트는 사용자 세션에 바인딩됨)
$principal = New-ScheduledTaskPrincipal -UserId $env:USERNAME -LogonType Interactive

Register-ScheduledTask -TaskName 'Measure KH' -TaskPath '\tsyi\' `
    -Trigger $triggers -Action $action -Settings $settings -Principal $principal -Force
```

설정값의 이유:

| 설정 | 값 | 이유 |
|------|----|------|
| `AllowStartIfOnBatteries` / `DontStopIfGoingOnBatteries` | 켬 | 노트북에서 기본값(배터리 시 중지)이 데몬·측정을 로그 없이 강제 종료하는 문제 회피 |
| `IdleSettings.StopOnIdleEnd` | `$false` | 유휴 종료 시 작업이 중단되지 않도록 |
| `ExecutionTimeLimit` | 55분 | 1회 측정(폭기 포함 약 37~44분) 여유 + 멈춘 측정 정리 |
| `MultipleInstances` | `IgnoreNew` | 직전 측정이 진행 중이면 다음 정시 트리거는 스킵 |
| `LogonType` | `Interactive` | COM(블루투스) 포트가 사용자 세션에 바인딩되어, 로그인된 대화형 세션에서만 접근 가능 |

> **로그인 세션 필요:** 로그아웃하면 세션·토큰이 파괴되어 작업이 죽습니다. 자리를 비울 땐 로그아웃 대신 **잠금(Win+L)** 을 쓰면 세션이 유지되어 계속 동작합니다(절전·디스플레이 OFF는 무방). 자동 복귀가 필요하면 자동 로그인을 권장합니다.
>
> 복잡한 PowerShell은 bat에 인라인으로 쓰지 말고 `.ps1`로 분리한 뒤 `register_measure_kh.bat`에서 `powershell -ExecutionPolicy Bypass -File "%~dp0register_measure_kh.ps1"` 한 줄로 호출합니다(한글 경로 + `^` 줄연속 조합 인코딩 오류 회피).

### 운용 메모

- 에어스톤 이전 직후 첫 사이클은 5L 첫 평형을 위해 폭기를 1~2시간 수동 연장
- 참조수 5L 증발 농축(1% ≈ +0.08 dKH 상당) → 주기적 RODI 수위 보충 또는 `calref`
- 3M KCl 제조: RODI 1L당 KCl 224g (≈22% w/v). 포화(≈4.2M@25°C) 대비 여유가
  있어 저온에서도 석출 없음
- 수동·단발 테스트는 펌웨어 `seq` 명령으로도 가능하나(user-manual.md 참조)
  수렴 판정은 지원되지 않음

## 온도 환경

위즈 탱크(5L) 안에 모든 비이커(200ml)가 담겨 있으므로:

- 참조수, 수조수 샘플, 3M KCl이 모두 동일 온도로 유지됨
- DS18B20이 위즈 탱크 수온을 측정하여 Nernst 온도 보상에 사용
- pH 전극의 온도 보상이 정확해짐

## 측정 정확도

### Nernst 온도 보정

pH 전극의 기울기는 온도에 따라 변합니다 (Nernst 방정식: slope = 0.19842 × T(K) mV/pH). 보정 온도를 EEPROM에 저장하여 보정 시와 측정 시 온도가 달라도 정확하게 보상합니다.

```
pH_corrected = 7.0 + (pH_raw − 7.0) × (273.15 + T_cal) / (273.15 + T_meas)
```

보정 온도 저장 후 **15°C~35°C 전 범위에서 Nernst 보정 오차 0%**입니다.

### 진단: 프로브 vs 셋업 분리 (널 테스트)

측정 시 참조수 pH와 수조수 pH 사이에 ΔpH 오프셋이 남을 수 있습니다(두 물이 동일 해수라면 이상적으로 ΔpH = 0). 이 오프셋의 원인이 **프로브·경로의 측정 아티팩트**인지, **두 물의 실제 화학 차이(CO2 탈기 비대칭)**인지 가르는 절차입니다.

**왜 재보정이 답이 아닌가:** dKH는 차동식(`KH_tank = KH_ref × 10^(−ΔpH)`)이라, pH 보정의 offset 오차는 참조·수조 양쪽에 똑같이 실려 ΔpH에서 상쇄되고, slope 오차도 작은 ΔpH(보통 0.02~0.07)에 비례하므로 영향이 미미합니다. 따라서 오프셋을 재보정(`calph`)으로 잡으려 하면 노이즈만 늘 뿐이며, 먼저 원인을 분리해야 합니다. (재보정은 절대 pH 동작점을 ADC 선형구간 안으로 유지하는 **예방·점검**용입니다.)

**절차:** 충분히 폭기(CO2 평형)한 **동일 해수**를 참조 경로와 수조 경로 양쪽에 넣고 1회 측정합니다.

| 결과 | 해석 | 조치 |
|------|------|------|
| ΔpH ≈ 0 | 오프셋 = 순수 화학(두 저장수의 CO2 상태 차이). 프로브·경로 정상 | 참조수 탈기 대칭 보강 (저장수조 직접 폭기, 폭기 시간·지속 폭기) |
| ΔpH ≠ 0 | 경로·접합·응답지연 아티팩트 | 프로브 응답속도(노화), 헹굼 대칭, 경로 부피·KCl 잔막 점검 |

**프로브 건강 계기:** 별도 장비 없이 `measure_kh.log`의 **수렴 반복 횟수**로 전극 상태를 모니터링합니다. 2~3회 이내로 빨리 수렴하면 정상, 횟수가 늘면 응답 둔화(노화) 또는 접합부(KCl junction) 오염 신호이며, 이때 묽은 HCl 접합 세척을 검토합니다. 접합전위 1 mV 변동 ≈ 0.017 pH ≈ 0.3 dKH이므로 무시할 수 없습니다.
