## 1. 시스템 개요

### 1.1 측정 원리

> 탈기 후 두 샘플의 CO2 농도가 동일해지면, pH 차이(DeltaPH)는 순수하게 알칼리니티(dKH) 차이만을 반영합니다.

```
[CO2] = 3 x KH x 10^(7-pH)
KH_tank = KH_ref x 10^(-DeltaPH)
DeltaPH = pH_ref - pH_tank
```

- `DeltaPH > 0` : 수조 pH가 참조보다 낮음 → 수조 dKH 낮음
- `DeltaPH < 0` : 수조 pH가 참조보다 높음 → 수조 dKH 높음

### 1.2 하드웨어 구성

| 부품 | 모델 | 역할 |
|------|------|------|
| MCU | Arduino Nano V3.0 (ATmega328P) | 메인 컨트롤러 |
| pH 센서 | DFRobot SEN0161-V2 | pH 전압 측정 |
| ADC | Adafruit ADS1115 | 16비트 I2C ADC (A4/A5) |
| 온도 센서 | DS18B20 | 1-Wire 온도 측정 (A0=D14) |
| 블루투스 | HC-06 | 하드웨어 Serial D0/D1 (9600baud) |
| 모터 드라이버 | L298N x 3 | L298N1/N2: 도징 펌프 4개 (12V), L298N3: 에어 펌프 (5V) |

### 1.3 핀 배치

| 핀 | 연결 | 설명 |
|----|------|------|
| D0 (RX) | HC-06 TX | 블루투스 수신 (직결) |
| D1 (TX) | HC-06 RX | 블루투스 송신 (전압분배기: R4 10k + R5 20k) |
| D4~D7 | L298N1 IN1~IN4 | 모터1/2 제어 |
| D8~D11 | L298N2 IN1~IN4 | 모터3/4 제어 |

## 2. 초기 설정

### 2.1 pH 2점 보정

> **주의:** 가장 먼저 수행해야 합니다. 보정값은 EEPROM에 저장되어 전원 OFF 후에도 유지됩니다.

| 단계 | 명령 | 설명 |
|------|------|------|
| 1 | `enterph` | 보정 모드 진입 |
| 2 | `calph` | pH 7.0 버퍼 용액 삽입 → 자동 인식 후 보정 |
| 3 | `exitph` | 보정값 EEPROM 저장 후 종료 |
| 4 | `enterph` | 다시 진입 |
| 5 | `calph` | pH 4.0 버퍼 용액 삽입 → 자동 인식 후 보정 |
| 6 | `exitph` | 보정값 EEPROM 저장 후 종료 |

### 2.2 온도 오프셋 설정

표준 온도계와 DS18B20의 차이값을 설정합니다. EEPROM에 저장됩니다.

```
settemp:-0.3
```

DS18B20 오프셋 설정 (예: 표준 25.0도, DS18B20 25.3도 → -0.3 입력). 범위: -10.0 ~ 10.0도.

설정 즉시 raw 온도와 보정 후 온도를 출력하여 확인할 수 있습니다.

### 2.3 참조 dKH 설정

참조 해수의 알칼리니티를 dKH로 입력합니다. EEPROM에 저장됩니다.

```
setref:8.5
```

참조 해수 dKH 설정 (범위: 0.5 ~ 30.0 dKH). 자연 해수 일반값은 약 7~8 dKH입니다. 실측값 사용을 권장합니다.

## 3. 측정 방법

### 3.1 표준 측정 절차

> **주의:** 참조수와 수조수를 동일 공간에서 30분 이상 동시 탈기 후 측정해야 정확합니다.

| 단계 | 명령 | 설명 |
|------|------|------|
| 1 | `settime:HH` | 현재 시(0~23) 설정 (이력 타임스탬프용, 예: `settime:14`) |
| 2 | `ref` | 참조수 pH 측정 (약 8초, 64회 오버샘플링) — RAM 저장 |
| 3 | `tank` | 수조수 pH 측정 (약 8초) |
| 4 | `calkh` | dKH 계산 + 결과 출력 + 이력 저장 |

> 참조수 pH는 RAM에만 저장됩니다. 전원 ON 후 매번 `ref` 명령을 실행해야 합니다.

### 3.2 자동 측정 (PC 연동)

> **변경(현재 펌웨어):** 폭기 중 측정을 반복해 **진짜 평형(평탄)까지** 측정해야 하므로(평탄 판정 = 측정값 비교 후 반복 결정), 자동화는 PC측 파이썬 스크립트가 담당합니다. 펌웨어의 `seq`/`seqstop`/`wait`/`air` 명령은 **제거**되었습니다. 단발 테스트는 위 §3.1처럼 `ref`/`tank`/`calkh`를 직접 입력하거나 모터/에어 명령을 수동 전송하면 됩니다.

PC측 자동 측정 스크립트:

```bash
# 1회 측정 후 종료 (Windows 작업 스케줄러 정시 호출 / 수동 실행 가능)
python bin/measure_kh_once.py        # 포트 = 기본값 COM9
python bin/measure_kh_once.py COM7   # 포트 직접 지정

# ref dKH 교정(역산) — 알려진 수조 dKH 로 참조수 dKH 를 거꾸로 보정
python bin/measure_kh_once.py --setref 8.448         # 수조 실측 8.448 dKH 기준
python bin/measure_kh_once.py COM7 --setref 8.448    # 포트 지정 + 교정
```

스크립트가 준비 → **[A] tank·ref 동시 폭기 + tank 평탄까지** → 전이 → **[B] ref 평탄까지** → `calkh` → 정리 순으로 펌웨어 명령을 단계별 전송합니다. 상세 흐름·상수·스케줄러 등록은 [자동화 환경 구성 — 자동 측정 시퀀스](system-setup.md#자동-측정-시퀀스-pc-측-v4)를 참조하세요.

**`--setref` (calref 모드, ref dKH 역산):** 인자를 주지 않으면 평소처럼 `calkh`(수조 dKH 측정 → `dkh.dat` 기록)를 수행합니다. `--setref <수조실측dKH>` 를 주면 동일한 tank·ref pH 측정 후 `calkh` 대신 **`calref`** 를 호출해 **참조수 dKH 를 역산·EEPROM 저장**합니다. 절차: ① 명령라인 수조 dKH 를 `setref` 로 기록(측정 전 범위 0.5~30.0 즉시 검증) → ② tank·ref pH 측정(동일) → ③ `calref` 가 `newRefDKH = 수조dKH · 10^(−(tankPH−refPH))` 를 계산하고 `refDKH` 에 대입 + EEPROM 저장(펌웨어가 자동 저장하므로 추가 `setref` 불요). 보정은 일회성이라 **`dkh.dat` 에는 기록하지 않습니다.** 위즈 참조수 같은 알려진 표준액으로 수조를 채워 측정하면, 그 값을 기준으로 참조수 dKH 앵커를 교정할 수 있습니다.

### 3.3 calkh 결과 출력

```
===[dKH]===
시각:14   참조pH:8.350   수조pH:8.201   dPH:0.1490   refKH:8.50 dKH   수조KH:6.012 dKH   온도:25.0C
===========
```

- ppm, meq/L, 상태 표시는 제거되었습니다. dKH 값으로 직접 판단하세요.
- 산호 수조 권장 범위: **8 ~ 12 dKH**

### 3.4 status 출력 예시

```
=== 상태 ===
시각:14
온도:25.30C 오프셋:-0.30 보정T:25.0C
수조pH:8.201
참조pH:8.350
dPH:0.1490
refKH:8.50 dKH
수조KH:6.012 dKH
KH이력:2개
[M1] 정지
[M2] 정지
[M3] 정지
[M4] 정지
[직접] 에어(D12):OFF PWM(D13):OFF
============
```

| 항목 | 설명 |
|------|------|
| 온도 / 오프셋 / 보정T | 현재 수온, DS18B20 오프셋, pH 보정 시 온도 (Nernst 기준) |
| [M1]~[M4] | 모터 상태 — 동작 중이면 잔여 시간 표시 |
| [직접] 에어(D12) / PWM(D13) | D12(에어 펌프)·D13(PWM 속도 제어기) 핀 ON/OFF 상태 |

> 구 펌웨어의 `[대기]`(wait)·`[SEQ]`(seq) 표시는 해당 명령 제거와 함께 사라졌습니다.

### 3.5 calref — 참조 dKH 역산

참조 해수의 dKH를 직접 측정하기 어려운 경우, 수조수의 dKH(상용 테스트킷 실측값)를 기준으로 참조수의 dKH를 역으로 계산합니다.

```
1. setref:8.5          ← 수조 dKH 실측값 (상용 테스트킷)
2. ref                 ← 참조수 pH 측정
3. tank                ← 수조수 pH 측정
4. calref             ← 참조 dKH 역산 + EEPROM 저장
```

**계산 공식:**

```
newRefDKH = 수조dKH × 10^(-(tankPH − refPH))
```

> 참고: calref의 `dPH`는 `tankPH − refPH`로, calkh/§1.1의 ΔpH(`refPH − tankPH`)와 부호가 반대입니다. calref는 calkh의 역산(참조 dKH를 거꾸로 구함)이라 부호가 뒤집힌 것이며, 결과는 서로 일관됩니다.

**출력 예시:**

```
===[calref]===
  시각:14
  참조pH:8.350
  수조pH:8.201
  dPH:-0.1490
  새refDKH:11.985 dKH
  수조dKH:8.50 dKH
  온도:25.0C
===============
[OK] refDKH 저장:11.985 dKH
```

> 이후 `calkh`로 측정하면 저장된 refDKH를 기준으로 수조 dKH를 자동 계산합니다. 이력은 저장하지 않습니다.

## 4. 명령어 전체 참조

### 4.1 pH 측정 명령

| 명령 | 설명 |
|------|------|
| `settime:HH` | 현재 시 설정 (0~23, 예: `settime:14`) — 이력 타임스탬프용 |
| `ref` | 참조수 pH 측정 → RAM 저장 (전원 OFF 시 소멸, 매번 측정 필요) |
| `tank` | 수조수 pH 측정 |
| `calkh` | dKH 계산 + 출력 + 이력 저장 (최대 5개) |
| `calref` | 수조 dKH 기준으로 참조 dKH 역산 + EEPROM 저장 (아래 참고) |
| `setref:xx.x` | 참조 dKH 설정 [EEPROM] (예: `setref:8.5`, 범위: 0.5~30.0) |
| `settemp:xx.x` | 온도 오프셋 설정 [EEPROM] (예: `settemp:-0.3`, 범위: +/-10.0) |
| `khhist` | dKH 이력 조회 (최신순 5개, 번호/시각/dKH) |
| `status` | 현재 전체 상태 출력 (아래 참고) |

| `help` | 명령어 도움말 출력 |

### 4.2 pH 보정 명령

| 명령 | 설명 |
|------|------|
| `enterph` | 보정 모드 진입 — 버퍼 용액에 프로브 삽입 후 안정화 |
| `calph` | pH 4.0 / 7.0 자동 인식 후 보정 |
| `exitph` | 보정값 EEPROM 저장 후 종료 |

### 4.3 펌프 모터 명령

| 명령 | 설명 |
|------|------|
| `m1f:초` | 모터1 정방향 N초 (예: `m1f:10`, 범위: 1~3600초) |
| `m1b:초` | 모터1 역방향 N초 |
| `m1s` | 모터1 즉시 정지 |
| `m2f:초` / `m2b:초` / `m2s` | 모터2 (L298N1 채널B) |
| `m3f:초` / `m3b:초` / `m3s` | 모터3 (L298N2 채널A) |
| `m4f:초` / `m4b:초` / `m4s` | 모터4 (L298N2 채널B) |

> PC측 측정 스크립트는 각 모터 명령의 완료 응답(`[모터N] 완료`)을 받은 뒤 다음 단계로 진행합니다.

### 4.4 에어 / PWM 공급 명령

| 명령 | 설명 |
|------|------|
| `ron` | D12 HIGH — 에어 펌프 ON (참조수+수조수 폭기) |
| `roff` | D12 LOW — 에어 펌프 OFF |
| `ton` | D13 HIGH — PWM 속도 제어기 ON (도징 펌프 구동 전 활성화) |
| `toff` | D13 LOW — PWM 속도 제어기 OFF |
| `airoff` | D12·D13 동시 LOW — 에어 + PWM 제어기 둘 다 OFF |

> `ron`/`roff`로 에어 펌프(D12)를, `ton`/`toff`로 PWM 속도 제어기(D13)를 독립적으로 켜고 끕니다. `airoff`는 두 핀을 한 번에 LOW로 내립니다.
> **하드웨어 규칙:** 액체 이동(`mXf`/`mXb`) 직전에는 반드시 `airoff`로 폭기를 끄고, 펌프 구동 전 `ton`으로 PWM 제어기를 켭니다.
> 구 펌웨어의 `air:총초:주기초`(교대 폭기) 명령은 **제거**되었습니다.

### 4.5 정지 명령

| 명령 | 설명 |
|------|------|
| `m1s`~`m4s` | 해당 모터 개별 즉시 정지 |
| `stop` | 전체 모터 정지 + 에어/PWM 핀 OFF (motorAllStop + stopAir) |

> 구 펌웨어의 `seq`/`seqstop`(시퀀스 매크로)·`wait:`(대기) 명령은 **제거**되었습니다. 자동 측정은 PC측 스크립트가 담당합니다(§3.2).

## 6. EEPROM 저장 항목

| 항목 | 저장 위치 | 비고 |
|------|-----------|------|
| pH 보정값 (pH4/7 전압) | `0x00~0x07` | DFRobot_PH 라이브러리 자동 관리 |
| 참조 dKH (`setref`) | `0x10~0x13` | 전원 OFF 후에도 유지 |
| 온도 오프셋 (`settemp`) | `0x14~0x17` | 전원 OFF 후에도 유지 |
| pH 보정 온도 (`exitph`) | `0x18~0x1B` | Nernst 보정 기준 온도, 보정 시 자동 저장 |
| 참조수 pH 전압 (`ref`) | RAM만 | 전원 ON 후 매번 측정 필요 |
| dKH 측정 이력 (`calkh`) | RAM만 | 전원 OFF 시 소멸 |

## 8. 사용 팁

### 8.1 정확한 측정을 위한 필수 조건

1. **밀폐 공통 헤드스페이스(필수):** 참조수 통·측정 챔버·펌프 흡입을 **하나의 밀폐 공기**로 묶습니다. ref·tank가 같은 한 덩어리 공기를 공유해야 측정 순간 pCO₂가 일치하고 방 CO₂ 변동이 차동으로 상쇄됩니다. **분리 밀폐 금지**(서로 다른 밀폐 공간은 따로 드리프트해 상쇄가 깨짐). 증발 차단 효과도 함께 얻습니다. → [밀폐 공통 헤드스페이스](system-setup.md#위즈-탱크)
2. **동시 폭기 + 평형까지 측정:** 참조수와 수조수를 같은 공기로 동시에 폭기하고, V4는 **폭기를 유지한 채** tank·ref가 각각 진짜 평형(평탄)에 도달할 때까지 반복 측정합니다. 밀폐로 평형 기준(헤드스페이스 pCO₂)이 고정돼 있어 "뚜껑 닫고 한 번 측정"이 아니라 **고정된 평형으로 끌고 가 공통모드 상쇄**가 깨끗하게 일어납니다.
3. **온도 보정:** `settemp`로 DS18B20 오프셋을 설정하면 pH 계산 정확도가 높아집니다.
4. **pH 보정:** 월 1회 pH 4.0/7.0 버퍼 용액으로 2점 보정을 실시합니다.
5. **ref 재측정:** 전원 ON 후 반드시 `ref` 명령으로 참조수를 다시 측정해야 합니다(수동 측정 시).
6. **업로드:** HC-06을 D0/D1에서 분리한 후 업로드하고, 완료 후 다시 연결합니다.

### 8.2 산호 수조 권장 dKH 범위

| 범위 | dKH | 비고 |
|------|-----|------|
| 자연 해수 | 약 6.5~7.5 | 참조 기준 |
| 산호 수조 권장 | 8 ~ 12 | 일반적 권장 |
| SPS 산호 최적 | 8 ~ 9 | 경산호류 |
| LPS/소프트 코랄 | 7 ~ 11 | 연산호류 |

## 10. 도저 자동 조정 (올포리프)

메인 수조(250L)의 dKH를 목표값으로 유지하기 위해, 측정 결과를 바탕으로 올포리프(AFR) 도징량 조정을 계산합니다(`bin/doser_adjust.py`).

### 10.1 동작 개요

- **매일 1회(13시 측정 후)** 권고를 계산합니다(2026-07-11부터 주말 포함). 현재 **자동 적용은 꺼짐(권고 전용)** — 계산·기록만 하고 도저 설정은 바꾸지 않습니다. 수동 설정·목표 저장은 이와 무관하게 계속 적용됩니다.
- 도저 하드웨어: 1회 도즈 = 펌프 8초 = 2mL, 하루 6회(4시간 간격). 통에는 **50% 희석액**을 담으므로 실도징(펌프가 뿜는 부피)은 원액 환산의 2배입니다.

### 10.2 조정 개념 — 오차와 속도

- **오차(수준):** 현재 dKH = **최근 유효 측정 3회의 중앙값**과 목표의 차이. 하루 3회 측정이라 정확히 24시간 창 — 야간 CO₂ 편향 같은 하루 주기 극단값 1개는 중앙값이 걸러냅니다.
- **속도(추세):** **최근 7일 21회 측정의 기울기**(Theil-Sen 중앙값 기울기, 튀는 값에 강함) — 지금 도징으로 dKH가 하루에 움직이는 속도입니다.
- **조정 원리:** 필요한 속도(오차÷7일)에서 현재 속도를 뺀 만큼만 도징량을 변경합니다 — 이미 목표 쪽으로 움직이고 있으면 그만큼 덜 바꿉니다(속도 보상).
- **목표 접근:** 오차를 **7일**에 걸쳐 좁히되, dKH 변화 속도 상한은 **±0.25 dKH/일**(허용 오차 ±0.5의 절반)입니다.

### 10.3 안전 한계

| 항목 | 값 |
|------|-----|
| 1회 조정 폭 | 현재 도징량의 최대 ±30% |
| 도징량 절대 범위 | 원액 1.5~18.0 mL/일 (실도징 3~36 mL/일, 펌프 2000~24000ms) |
| 데드밴드 | 원액 0.15 mL/일(200ms) 미만 차이는 유지 |
| 감도 가정 | 원액 1 mL/일 ≈ +0.006 dKH/일 (6/29 볼루스 55mL→+0.32 dKH 실측 유도) |
| 중단·보호 | 최근 7일 유효 측정 10점 미만이면 조정 안 함 · 적용 실패 시 이전값 롤백 · 수동 설정이 있는 회차는 자동 조정 생략 |

### 10.4 대시보드에서 수동 설정·목표 설정

- **수동 설정(원액 1.5~18 mL/일):** 도저 카드의 입력란에 값을 넣고 "수동 설정"을 누르면 GitHub에 커밋되고, 측정 PC의 폴러가 **최대 ~5분 내** 도저에 적용합니다. 수동 설정이 있는 회차는 자동 권고를 생략합니다.
- **목표 설정(6.0~9.0 dKH, 기본 7.2):** 저장 즉시가 아니라 **다음 자동 조정 회차(매일 13시)부터** 반영됩니다.
- 두 기능 모두 첫 사용 시 **GitHub 토큰**(fine-grained, 이 저장소 Contents 읽기/쓰기)을 입력해야 하며, 토큰은 해당 기기 브라우저에만 저장됩니다.
