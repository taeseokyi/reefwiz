# KH 측정 대장 (코드 상태 · 환경 · 결과)

ΔpH/dKH 검증을 계속하려면 "각 측정이 어떤 코드·환경에서 나왔는가"를 정확히 알아야 한다.
git 커밋만으로는 부족하다 — **6/16 13:00 변경(STABLE·EPS)과 v2 재작성은 커밋되지 않고 작업트리에서만
돌았다.** 그래서 이 대장에 측정 시각별로 직접 기록한다.

**기록 원칙:** 새 측정이 끝날 때마다 아래 표에 한 줄 추가. 값은 `measure_kh.log`(실제 실행 진실)와
`dkh.dat`에서 가져온다. 코드를 바꾸면 ① 새 "코드 버전" 항목을 정의하고 ② 가능하면 **커밋해 SHA를 남긴다**
(미커밋이면 표에 `(작업트리)`로 표기). 환경(방 CO₂)은 직접 로그가 없으므로 **절대 pH를 대리지표**로 쓴다
(절대 pH 낮음 = 방 CO₂ 높음). dKH 계산: `tank_dKH = 8.448 × 10^(−(ref_pH−tank_pH))`, ΔpH는 `tank_pH − ref_pH`.

dkh.dat 컬럼: `HH ref_pH tank_pH ref_kh(앵커 8.448) tank_kh temp`

## 현재까지의 핵심 관찰 (이 대장이 뒷받침하는 결론)

1. **새벽(05:00) 음 ΔpH는 코드 버전과 무관하게 방 CO₂ 높을 때 나타난다.** V1(6/16 05:00 −0.059)과 V2(6/17 05:00 −0.058)가 동일. v2(동시폭기·시간상수 일치 시도)도 못 잡음.
2. **측정 시간차는 원인이 아니다.** 연속 이틀 새벽 6/15 **+0.020**(V0) vs 6/16 **−0.059**(V1)로 부호까지 반대. V0와 V1은 **측정 시퀀스·시간차(~6.3분)가 동일**하고 측정 *사이* 지속폭기 유무만 다른데, 6/15 05:00은 사용자가 수동 ron으로 그 지속폭기를 대신 제공 → **두 새벽 모두 사전 연속폭기 상태로 동등.** 그런데도 부호가 뒤집힘 → 일정한 시간차도, 폭기 상태도 부호 반전을 만들 수 없음.
3. **ΔpH는 절대 pH(방 CO₂ 수준)와 상관한다.** 절대 pH 낮을수록(CO₂ 높을수록) ΔpH가 음으로 커짐. tank가 ref보다 방 CO₂에 더 빠르게 반응(평형)·ref는 지연(lag) = 부피·시간상수 비대칭이 유력 정체.
   - **자연 실험(환기) 정황:** 6/15 05:00은 **창문 열어둔(환기) 날** → 절대 pH ~8.07(CO₂ 낮음), ΔpH **+0.020**(정상). 닫힘방 새벽(6/16·6/17)은 절대 pH 7.78/7.97, ΔpH **−0.059/−0.058**. → "새벽 방 CO₂가 ΔpH 오차의 1차 인자"를 관측으로 뒷받침. (이 자연 실험으로 진단은 이미 충족 — 별도 환기 실험 불필요.)

## ★설계 원칙 / 해법 제약 (확정)

- **이 시스템은 환기와 무관하게 정확해야 한다. 환기는 해법이 아니다.** 방 CO₂는 차동(ref↔tank, 동일 프로브)이 **상쇄해야 하는 공통모드 변수**다. 지금 결함은 "방 CO₂가 ΔpH로 새어든다"는 것 자체이고, 그게 고칠 대상이다. 환기는 결함을 가려줄 뿐(6/15) 제거하지 않는다.
- **상쇄 조건:** 측정 순간 ref·tank가 **같은 pCO₂**여야 한다. 깨지는 이유 = ① ref·tank 시간상수(부피) 비대칭 → 새벽 CO₂ 급변 시 tank(빠름) 추종·ref(느림) 지연, ② 단일 프로브 순차 측정 → tank·ref 측정 시점 사이 방 CO₂ 변동.
- **해법 = 대칭 복원**(방 CO₂가 어떻든 ref·tank가 항상 같은 pCO₂가 되게):
  - **A.** ref를 tank와 **동일한 측정챔버·에어스톤에서 완전 co-equilibration**(v2는 ref를 홀딩챔버서 폭기 → 부피·효율 달라 평형 어긋남 가능성 직접 검정).
  - **C.** 참조수 **소부피화로 시간상수를 tank와 일치**(정공법, Fleger2025). 증발·앵커 드리프트 게이트 선결.
  - 남는 안정 baseline(~−0.02 ≈ −0.4 dKH)만 영점보정.
- **B(의도적 환기 실험)는 폐기** — 해법이 아니고, 진단도 6/15 자연실험으로 이미 됨.
4. 정상 시간대 baseline은 작고 안정(−0.008 ~ −0.028, 대표 ~−0.02 ≈ −0.4 dKH) → 영점보정 후보.

상세 진단·가설 이력은 메모리 `project-kh-offset-aeration-test` 참조.
