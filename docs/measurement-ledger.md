# KH 측정 대장 (코드 상태 · 환경 · 결과)

ΔpH/dKH 검증을 계속하려면 "각 측정이 어떤 코드·환경에서 나왔는가"를 정확히 알아야 한다.
git 커밋만으로는 부족하다 — **6/16 13:00 변경(STABLE·EPS)과 v2 재작성은 커밋되지 않고 작업트리에서만
돌았다.** 그래서 이 대장에 측정 시각별로 직접 기록한다.

**기록 원칙:** 새 측정이 끝날 때마다 아래 표에 한 줄 추가. 값은 `measure_kh.log`(실제 실행 진실)와
`dkh.dat`에서 가져온다. 코드를 바꾸면 ① 새 "코드 버전" 항목을 정의하고 ② 가능하면 **커밋해 SHA를 남긴다**
(미커밋이면 표에 `(작업트리)`로 표기). 환경(방 CO₂)은 직접 로그가 없으므로 **절대 pH를 대리지표**로 쓴다
(절대 pH 낮음 = 방 CO₂ 높음). dKH 계산: `tank_dKH = 8.448 × 10^(−(ref_pH−tank_pH))`, ΔpH는 `tank_pH − ref_pH`.

dkh.dat 컬럼: `HH ref_pH tank_pH ref_kh(앵커 8.448) tank_kh temp`

### ★통찰 — 6/16 vs 6/17 raw pH는 우연이 아니다 (2026-06-17)
```
6/16 05:00:  ref 7.810   tank 7.751
6/17 05:00:  ref 7.997   tank 7.939
  Δ(day→day): +0.187      +0.188      ← ① ref·tank 똑같이 상승 = 공통모드
  ΔpH(t−r):   −0.059      −0.058      ← ② 차동은 1 mpH로 동일
```
1. **공통모드 +0.188 → 차동 상쇄.** ref·tank가 정확히 같은 양 이동(전극 드리프트 또는 그날 밤 방 CO₂ 수준차 — 어느 쪽이든 공통모드) → ΔpH 보존. ⟹ **절대 pH는 날짜 간 "방 CO₂ 대리지표"로 부적합(공통모드 혼입). 깨끗한 신호는 ΔpH뿐.** (측정 기록표의 "절대 pH=CO₂ 대리"는 *같은 날 내*에서만 신뢰.)
2. **ΔpH가 코드(V1/V2)·간격(377/313s)·절대수준 다 달라도 1 mpH로 재현 = 안정적·결정론적 계통오차.** ⟹ ① 간격/시퀀스가 원인 아님을 강하게 보강, ② 재현되니 **영점보정 가능한 종류**(랜덤·드리프트 아님).
3. **(가설) ΔpH가 CO₂ 스윙 크기에 비례 안 함 → dC/dt 모델 흔들림.** 밤샘 축적 정황: 6/15 21:00(8.10)→6/16 05:00(7.78) ≈ −0.32(큰 축적) vs 6/16 21:21(8.03)→6/17 05:00(7.97) ≈ −0.06(작은 축적)인데 ΔpH 동일. "ΔpH≈간격×dC/dt"면 6/17(간격 짧음)이 더 작아야 하나 동일 → dC/dt가 마침 20%↑여야 상쇄(부자연). ⟹ **새벽 ΔpH는 rate 기반 과도가 아니라 닫힘(고-CO₂) 조건의 포화/구조적 비대칭** 쪽. (※절대 pH 기반이라 드리프트 혼입 가능, 확정 아닌 가설.)
4. **검정 제안(순수 진단, 해법 아님):** 방을 *스윙 없이 수준만* 고-CO₂로 안정시켜 측정 → 그래도 −0.058이면 "수준 의존 구조 비대칭" 확정(rate 아님).

**해석 (정직 — 이 세 점은 간격 단독 효과를 못 가른다):**
- **6/15 ≡ 6/16: 간격 완전 동일(377s)인데 ΔpH +0.020 vs −0.059.** 간격이 같으니 차이는 순전히 환경(dC/dt, 환기 유무). 깨끗한 대조 → "ΔpH ≈ 간격 × dC/dt"의 dC/dt 항을 입증.
- **6/16 vs 6/17: 간격 377→313s(~17%↓)인데 ΔpH −0.059→−0.058 거의 불변.** 표면상 "간격 줄여도 효과 없음" = **간격이 레버가 아님** 쪽 시사. 단 다른 밤(dC/dt 다를 수 있음)·다른 코드라 간격 단독 비교 불가.
- **결론: 세 점만으론 간격(timing) 단독 효과 분리 불가.** 진짜 검정 = **같은 밤(같은 dC/dt)에 간격/순서만 바꿔 연속 측정.**
- 보조 단서: 세 세션 모두 수렴 2회 연속 **Δ=0.0000(45초 간격)** → ~1분 스케일에선 시료 표류 없음. 분 단위 간격차가 −0.058을 만든다는 그림과 다소 어긋남 → 밀봉/안정가스 가설 쪽에 무게.

## 현재까지의 핵심 관찰 (이 대장이 뒷받침하는 결론)

1. **새벽(05:00) 음 ΔpH는 코드 버전과 무관하게 방 CO₂ 높을 때 나타난다.** V1(6/16 05:00 −0.059)과 V2(6/17 05:00 −0.058)가 동일. v2(동시폭기·시간상수 일치 시도)도 못 잡음.
2. **측정 간격 "크기"는 day-to-day 변수가 아니다(단, 간격은 누출 통로다 — 위 설계 원칙의 정정 참조).** 연속 이틀 새벽 6/15 **+0.020**(V0) vs 6/16 **−0.059**(V1)로 부호까지 반대. V0·V1은 측정 시퀀스·간격(~6.3분) 동일, 6/15는 수동 ron으로 사전폭기까지 동등. 그런데 부호 반전 → 변한 건 간격·폭기상태가 아니라 **방 CO₂ 변화율(dC/dt)**. 정확한 관계 **ΔpH ≈ 간격 × dC/dt**: 간격 일정 → 변동은 dC/dt 몫(6/15 환기 dC/dt≈0, 6/16 닫힘 dC/dt 큼).
3. **ΔpH는 절대 pH(방 CO₂ 수준)와 상관한다.** 절대 pH 낮을수록(CO₂ 높을수록) ΔpH가 음으로 커짐. tank가 ref보다 방 CO₂에 더 빠르게 반응(평형)·ref는 지연(lag) = 부피·시간상수 비대칭이 유력 정체.
   - **자연 실험(환기) 정황:** 6/15 05:00은 **창문 열어둔(환기) 날** → 절대 pH ~8.07(CO₂ 낮음), ΔpH **+0.020**(정상). 닫힘방 새벽(6/16·6/17)은 절대 pH 7.78/7.97, ΔpH **−0.059/−0.058**. → "새벽 방 CO₂가 ΔpH 오차의 1차 인자"를 관측으로 뒷받침. (이 자연 실험으로 진단은 이미 충족 — 별도 환기 실험 불필요.)

## ★설계 원칙 / 해법 제약 (확정)

- **이 시스템은 환기와 무관하게 정확해야 한다. 환기는 해법이 아니다.** 방 CO₂는 차동(ref↔tank, 동일 프로브)이 **상쇄해야 하는 공통모드 변수**다. 지금 결함은 "방 CO₂가 ΔpH로 새어든다"는 것 자체이고, 그게 고칠 대상이다. 환기는 결함을 가려줄 뿐(6/15) 제거하지 않는다.
- **상쇄 조건:** 측정 순간 ref·tank가 **같은 pCO₂**여야 한다.
- **★v2는 이미 소량 동일 부피(사용자 확인 2026-06-17):** ref·tank 모두 60초 펌프 분취(소량)를 직접 폭기·측정. 즉 **부피·시간상수 일치(C)는 v2에서 사실상 구현됨.** 그런데도 6/17 05:00 −0.058 → **"부피·시간상수 비대칭"은 v2 잔여 오차의 원인이 아니다(기각).** (V0/V1은 ref를 5L 저장조에서 떠와 lag 소지가 있었으나, v2는 소량 분취 직접 폭기로 그걸 없앰.)
- **소거법(전부 사용자 확인 2026-06-17):** ① 부피·시간상수 — v2에서 이미 동일(소량 분취). ② 챔버·에어스톤 — **이미 동일**. 둘 다 기각. → 폭기 종료 시점엔 ref·tank가 동일 pCO₂.
- **★잔여 단일 원인 = 단일 프로브의 순차 측정 시점차.** tank(t1)·ref(t2, ~5분 뒤)가 *변하는* 방 공기를 향해 다른 시각에 재평형 → 어긋남.
  - **정정: "측정 시간차는 원인 아님"(아래 #2)은 부정확.** 정확히는 **ΔpH ≈ (측정 간격) × (그 사이 방 CO₂ 변화율 dC/dt).** 간격=통로, dC/dt=곱해지는 양. 간격이 ~6분으로 *일정*했기에 날짜별 변동은 dC/dt가 만들었을 뿐(6/15 환기 dC/dt≈0→+0.020 / 6/16·17 닫힘새벽 dC/dt 큼→−0.058). 간격 자체는 여전히 누출 통로.
- **★★물리 셋업 정정 (사용자 2026-06-17):** 측정 대상 두 물의 *소스 용기*가 다르다 —
  - **ref = 5L 김치통**(반밀폐, 측정챔버들이 잠긴 온도욕조 겸 참조수 소스).
  - **tank = 별도 200mL 외부 챔버, 외기에 *개방***(수조 시뮬레이터). ← 본수조=라이브 리프 아님, 200mL 개방 컵.
  - 둘 다 6/14 20:12 같은 물 분취(널테스트, 진짜 ΔKH=0).
  - 측정챔버(프로브 읽는 곳)는 통에 잠겨 한 펌프(방 공기)로 1200s 폭기.
- **★−0.058 정체 = 200mL vs 5L 시간상수 비대칭(원래 가설이 옳았다).** 개방 200mL(작음)=방 CO₂ 빠른 추종 / 반밀폐 5L(25배)=느린 lag. 새벽 방 CO₂↑ → tank(200mL) 고-CO₂·낮은 pH(7.939) / ref(5L) 지연·높은 pH(7.997) → ΔpH −0.058. **메모리 원래 "tank 빠른추종/ref 느린 lag" 모델 그대로.** ※앞서 "부피 기각"은 *측정 분취*만 보고 한 오판 — 비대칭은 *소스 용기*(200mL 개방 vs 5L 반밀폐)에 있었음.
- **왜 차동이 못 지우나 = 폭기 불완전.** 원리상 **완전 평형이면 부피 무관**([[reference-aeration-equilibrium]]) → 양쪽 같은 pCO₂로 완전 폭기하면 ΔpH=0. −0.058 잔류 = 1200s가 소스의 큰 출발 CO₂ 차이를 완전히 못 지움. ⟹ **소스 시간상수 비대칭(200/5L)이 새벽 큰 출발차 생성 + 1200s 폭기 미완 → 잔류 −0.058.** 통합 설명: 방CO₂ 펌프 공통(상쇄)·간격무관·챔버대칭·재현성(시간상수=물리상수)·6/15 환기/수동ron 도움 전부 정합.
- **레버 = 폭기 완전성.** ★검정(단일변경): **닫힘 새벽에 폭기시간 크게 연장**(1200→2400/3600s). ΔpH가 baseline(~−0.02)으로 줄면 **폭기 미완 확정 + 해법 확정**(완전평형=부피·소스·방CO₂ 전부 무관=환기 무관 로버스트). 그대로면 폭기로 안 지워짐 → **ref 소부피화로 소스 시간상수 직접 일치** 필요.
- 남는 안정 baseline(~−0.02 ≈ −0.4 dKH)만 영점보정. **B(방 환기) 폐기.** ※200mL 개방 시뮬은 실제 큰 수조보다 방CO₂에 훨씬 민감 → 이 널테스트는 worst-case 스트레스.
- 정상 시간대 baseline은 작고 안정(−0.008 ~ −0.028, 대표 ~−0.02 ≈ −0.4 dKH) → 영점보정 후보.

## 프로브 보정 상태 (provenance)

- **2점 보정(calph 4.01 / 7.0) 미실시** (2026-06-17 기준). 프로브 = DFRobot SEN0161-V2, ADC = ADS1115. EEPROM slope/offset은 과거 보정값 그대로.
- **측정 기울기(slope %): (미측정)** — `enterph` → pH4.01·pH7.0 버퍼 → `calph` → `exitph`로 읽어 여기에 기록 예정. Nernst 이상치(약 59 mV/pH @25℃) 대비 % = 프로브 건강 지표(95~100% 정상, 떨어지면 노화).
- **보정이 dKH에 (거의) 무영향인 이유 — 차동식 상쇄:** `tankDKH = 8.448 × 10^(−(refPH−tankPH))`, ΔpH에만 의존. 같은 프로브 하나가 ref·tank 순차 측정.
  - **offset 오차 → 완전 상쇄(영향 0):** ref·tank에 동일하게 실려 ΔpH에서 소거. (7.0 버퍼 보정이 틀려 있어도 dKH 무영향.)
  - **slope 오차 → ΔpH를 비율 스케일만:** `ΔpH_측정 = (S_실제/S_가정) × ΔpH_참`. 10% 오차 시 → 새벽 ΔpH 0.058에서 dKH **~0.1**, baseline에서 **~0.04**. **기존 ΔpH를 ±몇 %로 늘릴 뿐, 새벽 dip을 생성·설명하지 못함.**
- **함의:** 보정 미실시는 우리가 쫓는 새벽 ΔpH 아티팩트의 원인이 **아니다**(차동 상쇄). 보정은 **수정 도구가 아니라 진단·예방용** — ① 기울기% 건강검진(크게 떨어졌으면 전체 dKH 스프레드에 계통 gain 오차) ② 동작점/선형구간 리셋(절대 pH 표류로 ADS1115 입력범위·선형 Nernst 구간 이탈 방지). 절대 pH를 방 CO₂ 대리지표로 쓰는 데도 보정이 맞으면 더 의미 있음(상대 추세는 미보정으로도 유효).
- **권고:** `calph` 4.01/7.0 1회 실행 → 기울기% 이 항목에 기록(건강 지표). 새벽 ΔpH 거동은 바뀌지 않음.
