# KH 측정 대장 (코드 상태 · 환경 · 결과)

> 관련 문서: [프로젝트 개요 (README)](../README.md) | [자동화 환경 구성](system-setup.md) | [사용 설명서](user-manual.md) | [준비물 목록](parts-list.md)

> **이 문서의 의의:** 측정 정확도의 신뢰성을 떠받치는 1차 기록이다. 코드 버전(V0~V4)별 실측, 환경(방 CO₂) 통제, 차동 측정이 방 CO₂ 의존성을 제거함을 보인 ΔpH 검증, 그 해석을 뒷받침한 학술 문헌까지 — 많은 시행착오의 결과를 시각순으로 남긴다.

*실시간 그래프(하루 3회 측정 자동 반영)는 [대시보드](https://taeseokyi.github.io/reefwiz/)에서 확인. 전체 이력 원본은 [`data/dkh.dat`](../data/dkh.dat).*

ΔpH/dKH 검증을 계속하려면 "각 측정이 어떤 코드·환경에서 나왔는가"를 정확히 알아야 한다.
git 커밋만으로는 부족하다 — **6/16 13:00 변경(STABLE·EPS)과 v2 재작성은 커밋되지 않고 작업트리에서만
돌았다.** 그래서 이 대장에 측정 시각별로 직접 기록한다.

**기록 원칙:** 새 측정이 끝날 때마다 아래 표에 한 줄 추가. 값은 `measure_kh.log`(실제 실행 진실)와
`dkh.dat`에서 가져온다. 코드를 바꾸면 ① 새 "코드 버전" 항목을 정의하고 ② 가능하면 **커밋해 SHA를 남긴다**
(미커밋이면 표에 `(작업트리)`로 표기). 환경(방 CO₂)은 직접 로그가 없으므로 **절대 pH를 대리지표**로 쓴다
(절대 pH 낮음 = 방 CO₂ 높음). dKH 계산: `tank_dKH = 8.448 × 10^(−(ref_pH−tank_pH))`, ΔpH는 `tank_pH − ref_pH`.

dkh.dat 컬럼: `HH ref_pH tank_pH ref_kh(앵커 8.448) tank_kh temp`
- **에러 표식**: 5개 값 전부 `0.000…0.0` = 측정 실패/타임아웃/KCl 소크 실패. **에러 래치**(V4): 마지막 줄이 에러 표식이면 다음 실행은 측정 없이 에러 표식만 재기록 — *수동으로 마지막 에러 줄 제거* 전까지(오류 무인 반복측정 방지).
- **평탄 미도달 표식**(V4): `tank_kh`가 **음수(−)** = 정상 측정이나 평형 미도달(값 크기 유효, 서버가 부호로 인지). 에러 표식(전부 0)과 구분되어 래치 안 걸림.

## ★★★ 최종 결론 (2026-06-21) — 목표 정확도 달성, 계측 offset −0.06

며칠간의 "널테스트 오차가 계측기 결함인가, 물 문제인가" 모호함이 **완전 널테스트**로 종결되었다.

**핵심 결과 (완전 널, 6/21 19:00):** 위즈(5L) 해수를 떠서 tank 외부 챔버 100mL에도 넣어 **ref·tank 양쪽을 같은 물**로 만들었다(물 차이 = 구성상 0). 양쪽 모두 진짜 평탄 도달(tank 21회·ref 18회 O), 온도 27.4℃ 안정:

| 측정 | ΔpH | dKH | 무보정 오차 (vs 8.448) |
|------|-----|-----|------------------------|
| **완전 널 (양쪽 같은 물)** | **−0.003** | **8.387** | **−0.061** |

**−0.92 절대오차의 정체 완전 규명:**
- **−0.86 dKH = 옛 200mL tank 시료 물의 알칼리니티 손실** (6일 개방+반복 폭기로 CaCO₃ 석출/생물막 → 알칼리니티 감소). *증발은 KH를 올리지만(밀폐가 차단), 폭기가 CaCO₃ 과포화를 키워 석출시키는 손실이 더 컸다.*
- **−0.06 dKH = 계측기 순수 시스템 offset** (물 차이 0에서 측정 → ΔpH −3mpH = 분해능/전극 미세 비대칭 수준).

**입증 경로 (부분 널 → 완전 널):**
```
옛 tank 물 (6/20~21):      ΔpH −0.05   dKH 7.4~7.6   오차 −0.9   ← 옛 시료 물 degrade
신선 tank vs 옛 5L (13:00): ΔpH −0.008  dKH 8.298     오차 −0.15  ← tank만 신선으로 교체
완전 널, 양쪽 같은 물(19:00): ΔpH −0.003  dKH 8.387     오차 −0.06  ← 물 차이 0 = 순수 offset
```

**목표 판정 ([README](../README.md) 사양 ±0.07, 사용자 희망 ±0.25 / 허용 ±0.5):**
- 정밀도(흩어짐): SD ±0.08 dKH → 합격
- 계측 offset: −0.06 dKH → ±0.25의 1/4, **영점보정 불요**(보정 가치 없음)
- ⟹ **희망 ±0.25 dKH 달성 확정.** 실제 리프수조의 정상 물이면 무보정 ±0.1 안.

**이를 가능케 한 3대 조건:**
1. **밀폐 공통 헤드스페이스** — ref·tank 같은 pCO₂ 공유(방 CO₂ 공통모드 상쇄) + 증발 차단.
2. **완전평형 측정** — 4시간 상한 + B1 평탄 게이트(span4 AND net8≤1)로 양쪽이 *진짜* 대기평형 천장까지. ⟹ 부피 비대칭(100mL vs 5L)도 완전평형에선 상쇄됨을 실측 확인.
3. **차동식** `KH_tank = 8.448 × 10^(ΔpH)` — 절대 pCO₂가 공통모드로 소거.

상세 시각순 기록은 아래 측정 기록표, 손실 메커니즘은 본 문서 "★석출(CaCO₃) 가설" 섹션 참조.

## ★★★ 며칠간 동일시료 완전 널 캠페인 통계 (2026-06-23 05:00 ~ 06-27 05:00, n=13)

6/22 22:00 시료 신선화(대용량 마개 외부챔버 + 해수 보충) 이후, **장치·시료 무변경으로 4일간 1일 3회(05/13/21시) 동일 시료 완전 널**을 연속 측정했다. 참값 = ref 앵커 **8.448 dKH**(같은 물 → 차동식상 출력이 앵커와 같아야 함, 편차 = 순수 계측 오차).

**통계 요약 (n=13, 출력 dKH):**

| 지표 | 값 | 판정 |
|------|-----|------|
| 평균 출력 | **8.318 dKH** | (참값 8.448) |
| **편향 (bias)** | **−0.130 dKH** | \|편향\| ≤ 0.25 → 희망목표 충족 |
| 편향 95% CI | **[−0.167, −0.093]** | **0 미포함 → 편향 실재(통계적 유의)** |
| **정밀도 σ** | **0.061 dKH** | 희망 ±0.25의 1/4 → 우수 |
| RMSE (총오차) | **0.142 dKH** | ±0.25 안 |
| 평균 95% CI | 8.318 ± 0.037 | — |
| **단발 1회 95% 예측구간** | −0.130 ± 0.120 = **[−0.25, −0.01]** | 아무 1회나 95% 확률로 참값 ±0.25 안 |
| ΔpH(tank−ref) 평균 | **−6.7 mpH** (SD 3.1) | ref>tank 계통 = 액간전위/전극 비대칭 |

**핵심 결론:**
1. **−0.13 dKH의 계통 편향이 실재한다** (95% CI가 0을 배제). 6/21 완전 널의 −0.06보다 약 2배 큰 값으로, **"−0.06·영점보정 영구 불요"(6/21 결론)는 이 캠페인으로 갱신됨** — 잔여 baseline이 더 크다.
2. **★온도·KCl은 원인이 아님 (사용자 결론).** 편향 vs 수온 상관계수 **r = −0.10**(≈0, 25.1~27.6℃ 범위 내 무상관)으로 온도 배제가 데이터로도 정합. KCl도 배제됨. ⟹ **남은 유력 원인 = 액간전위/기준전극 비대칭**(ΔpH가 일관되게 tank<ref −6.7mpH인 계통 offset, [project-kh-offset-aeration-test](notes/project-kh-offset-aeration-test.md)의 정적 baseline 항).
3. **그래도 목표는 충족**: 편향 −0.13 + 정밀도 σ 0.06 → RMSE 0.14, 단발 예측구간도 ±0.25 안. 희망 ±0.25 / 허용 ±0.5 모두 합격(영점보정으로 −0.13 제거하면 ±0.06까지 가능하나 운영상 불필요).

**신뢰수준 / 필요 표본수 (널 테스트라 σ 기지):**
- 편향 추정 95% CI = ±1.96σ/√n. **±0.25 보장 n≥1, ±0.1 보장 n≥2, ±0.05 보장 n≥6.**
- 현재 n=13으로 편향은 이미 ±0.037(95%)까지 좁혀짐 → **신뢰수준 확보 목적의 추가 측정 불필요.** 성능은 통계적으로 충분히 특성화됨.
- 정밀도를 더 줄이는 건 횟수가 아니라 물리(전극 드리프트·액간전위) 문제 = 단발 ±0.1 바닥([project-accuracy-target](notes/project-accuracy-target.md) 6/26).

**밀폐 챔버 온도 정정:** 이 구간 온도 28.9~29.9℃는 dkh.dat 온도컬럼이 **완전 밀폐형 측정챔버 내부에서 평형된 수온**이기 때문(리프위즈 박스 자체가 완전밀폐라 승온, 사용자 확인). 메인 리프탱크의 실제 수온 대리값이 아니며, ref·tank가 같은 온도를 공유해 dKH 계산(차동식)엔 무관함. [project-dkh-temp-column-is-sealed-box](notes/project-dkh-temp-column-is-sealed-box.md)

## ★석출(CaCO₃) 가설 — "옛 tank 물 dKH가 *왜 낮은가*"의 메커니즘 (2026-06-21)

사용자 지적: 분취 해수가 시간 지나면 **증발 농축 → dKH↑** 가 정상인데, tank는 ref보다 **낮다**(누적 7점 ΔpH −0.044~−0.056, tank≈7.926/ref≈7.976). 왜 내려가나?

**답: 개방·소부피·반복폭기 시료엔 KH 올리는 항(증발)과 내리는 항(석출)이 경쟁하고, 측정이 내려간 것 자체가 석출이 이겼다는 증거.**

- **증발 농축 → dKH↑** (이온 잔류). 단 **밀폐가 이 항을 거의 꺼버림**(6/19 검증 ②: 새벽→낮 dKH 점프 개방 +0.29 → 밀폐 −0.03, ~10배↓). ⟹ KH 올리는 항이 억제되어 내리는 항이 노출.
- **CaCO₃ 석출 → dKH↓**(알칼리니티 *제거*). 해수는 CaCO₃ 과포화(Ω_arag ~3–4). 측정마다 **폭기로 CO₂ 스트리핑 → pH 상승 → 과포화 더 상승 → 벽면·에어스톤·생물막에 CaCO₃ 석출**: `Ca²⁺ + 2HCO₃⁻ → CaCO₃↓ + CO₂ + H₂O`(알칼리니티 2당량 제거). **폭기 자체가 석출 촉진.** 200mL 소부피(높은 표면적/부피)·6일·수십 회 폭기 왕복 → 누적. 폭기 해수 시료의 알려진 알칼리니티 하락 아티팩트.
- **확정 아님 — 후보 3:** ① CaCO₃ 석출(물 degrade) ② 생물막 ③ 계통 offset(물 아니라 계측/전극/200mL↔5L 비대칭). 신선 동일 해수 널이 가름(ΔpH→0이면 ①②, 여전히 −0.05면 ③).
- **추가 확인 2:** (a) 200mL tank 통 벽면·에어스톤 흰 CaCO₃ 스케일 육안 점검(끼었으면 석출 직접 증거). (b) Salifert/Hanna 독립 KH 키트로 tank·ref 물 따로 측정 교차검증(tank<ref면 물 차이 확정, 같으면 계통 offset).

(a)장치 구조 오프셋이 아니라 (b)물/증발/전극 드리프트 성분이 큼이 드러남 → 지금 영점보정하면 망침. 보정값 펌웨어/서버 미적용 유지.

### 이 분석을 뒷받침한 참조 문헌 (논문 + 요약 + 어떤 해석에 쓰였나)

1. **Fleger, Liu, Berelson, Adkins 외 (2025), *Total alkalinity measurements in small samples: methods based on CO₂ equilibration and spectrophotometric pH*, Analytica Chimica Acta** — DOI 10.1016/j.aca.2025.344432.
   - 요약: 총알칼리도(AT)를 **AT·pCO₂·분광pH의 단순 선형관계**로 결정. 온도보정상수 E(T)를 쓰면 절대 CO₂ 농도가 불필요 — 즉 **ref·tank를 같은 조건으로 폭기하면 절대 pCO₂가 상쇄**된다. 직접 버블링으로 **0.5mL 소부피**까지, 정밀도 ±1~2 µmol/kg.

3. **Johnson (1982), *CO₂ hydration/dehydration kinetics in seawater*, Limnol. Oceanogr. 27(5):849** — DOI 10.4319/lo.1982.27.5.0849. (보강: Soli & Byrne; 이론서 Zeebe & Wolf-Gladrow 2001, *CO₂ in Seawater*, Elsevier Ocean. Ser. 65.)
   - 요약: 물속 CO₂ 내부 화학평형 완화시간 **≈16초**(25°C, S=35, pH8.2). 율속단계=CO₂ 수화 k≈0.037 s⁻¹.
   - 이번 쓰임: 평형의 율속이 **화학(16초)이 아니라 물리 가스교환(분 단위)**임을 명확히. 6/18 tank 곡선이 수십 분에 걸쳐 단조 이동한 것은 화학완화가 아니라 가스교환(부피·계면적 의존)임 — 아래 4번의 부피 비대칭 해석으로 연결.

4. **부피 ↔ 평형시간 스케일링** (가스스트리핑·kLa 문헌 종합, 2026-06-16 검색): ScienceDirect S0045653512015184, Roberts 2005 GRL, NIST(가스스트리핑); AIP Phys. Fluids·BioProcess Intl(kLa·VVM); Egleston 2010 GBC(Revelle); Zeebe & Wolf-Gladrow(평형시간).
   - 요약: 폭기 평형은 1차 지수 `C(t)−C* = (C₀−C*)e^(−t/τ)`, **시간상수 τ ∝ V/(K_H·Q_gas)·β**, 지배 무차원군 = **VVM(Q_gas/V)**. 같은 폭기유량이면 **τ ∝ V**(기포포화 영역) ~ 계면적/부피 a 의존(액막 영역). β=해수 CO₂ 버퍼(Revelle ~8~15배)는 τ를 곱으로 늘리나 V/Q 스케일링 보존.
   - 이번 쓰임: **"새벽 −0.058 문제 = 레벨 효과"** 해석의 정량 근거. tank=200mL(작은 V→짧은 τ→방CO₂ 빠른 추종) vs ref=5L(25배 V→긴 τ→느린 lag)의 시간상수 비대칭이, 불완전 평형 시 고-CO₂ 천장에 한쪽만 먼저 도달시켜 음의 ΔpH를 만든다. V4 완전평형은 양쪽을 같은 천장까지 끌고 가 이 비대칭을 제거 → 6/18 개선(−0.043) 설명.

5. **대기평형 = pH 천장 / 부피 무관 (원리, [reference-aeration-equilibrium](notes/reference-aeration-equilibrium.md) 정리)** — Henry 법칙([CO₂]aq=K_H·pCO₂는 부피 항 없음) + 평형 pH=f(AT, pCO₂, T, S).
   - 요약: 폭기는 물 pCO₂를 방 공기와 평형시키는 과정 → **평형 도달 후엔 더 폭기해도 pH 불변(대기평형이 천장)**, "과폭기" 오차는 없음. 평형 pH에 **부피 항이 없어**, 둘 다 완전평형이면 100mL vs 5L처럼 부피가 달라도 ΔpH 오차 0(부피 상쇄). 관건은 부피 일치가 아니라 **둘 다 완전평형 + ref·tank pCO₂·T 일치**.
   - 이번 쓰임: 6/18의 가장 중요한 해석 — **"완전평형이면 부피·소스·방CO₂ 전부 무관"**의 직접 근거. 새벽 tank가 7.960→7.695로 **하강**(천장이 출발 pH보다 아래)한 극단에서도 차동이 성립한 것, 그리고 −0.058→−0.043 개선이 "불완전→완전 평형" 전환으로 설명되는 것이 모두 이 원리. **단, 남는 −0.042가 "진짜 ΔKH=0인데 남은 장치 baseline"인지의 판정은 문헌이 아니라 새 널테스트 실측의 몫**(원리상 완전평형이면 0이어야 하므로, 잔류는 장치·증발 confound 후보).

### ★통찰 — 6/16 vs 6/17 raw pH는 우연이 아니다 (2026-06-17)
```
6/16 05:00:  ref 7.810   tank 7.751
6/17 05:00:  ref 7.997   tank 7.939
  Δ(day→day): +0.187      +0.188      ← ① ref·tank 똑같이 상승 = 공통모드
  ΔpH(t−r):   −0.059      −0.058      ← ② 차동은 1 mpH로 동일
```
1. **공통모드 +0.188 → 차동 상쇄.** ref·tank가 정확히 같은 양 이동(전극 드리프트 또는 그날 밤 방 CO₂ 수준차 — 어느 쪽이든 공통모드) → ΔpH 보존. ⟹ **절대 pH는 날짜 간 "방 CO₂ 대리지표"로 부적합(공통모드 혼입). 깨끗한 신호는 ΔpH뿐.** (측정 기록표의 "절대 pH=CO₂ 대리"는 *같은 날 내*에서만 신뢰.)

"ΔpH≈간격×dC/dt"면 6/17(간격 짧음)이 더 작아야 하나 동일 → dC/dt가 마침 20%↑여야 상쇄(부자연). ⟹ **새벽 ΔpH는 rate 기반 과도가 아니라 닫힘(고-CO₂) 조건의 포화/구조적 비대칭** 쪽. (※절대 pH 기반이라 드리프트 혼입 가능, 확정 아닌 가설.)
4. **검정 제안(순수 진단, 해법 아님):** 방을 *스윙 없이 수준만* 고-CO₂로 안정시켜 측정 → 그래도 −0.058이면 "수준 의존 구조 비대칭" 확정(rate 아님).

- **6/15 ≡ 6/16: 간격 완전 동일(377s)인데 ΔpH +0.020 vs −0.059.** 간격이 같으니 차이는 순전히 환경(dC/dt, 환기 유무). 깨끗한 대조 → "ΔpH ≈ 간격 × dC/dt"의 dC/dt 항을 입증.

## 현재까지의 핵심 관찰 (이 대장이 뒷받침하는 결론)

1. **새벽(05:00) 음 ΔpH는 코드 버전과 무관하게 방 CO₂ 높을 때 나타난다.** V1(6/16 05:00 −0.059)과 V2(6/17 05:00 −0.058)가 동일. v2(동시폭기·시간상수 일치 시도)도 못 잡음.
2. **측정 간격 "크기"는 day-to-day 변수가 아니다(단, 간격은 누출 통로다 — 위 설계 원칙의 정정 참조).** 연속 이틀 새벽 6/15 **+0.020**(V0) vs 6/16 **−0.059**(V1)로 부호까지 반대. V0·V1은 측정 시퀀스·간격(~6.3분) 동일, 6/15는 수동 ron으로 사전폭기까지 동등. 그런데 부호 반전 → 변한 건 간격·폭기상태가 아니라 **방 CO₂ 변화율(dC/dt)**. 정확한 관계 **ΔpH ≈ 간격 × dC/dt**: 간격 일정 → 변동은 dC/dt 몫(6/15 환기 dC/dt≈0, 6/16 닫힘 dC/dt 큼).
3. **ΔpH는 절대 pH(방 CO₂ 수준)와 상관한다.** 절대 pH 낮을수록(CO₂ 높을수록) ΔpH가 음으로 커짐. tank가 ref보다 방 CO₂에 더 빠르게 반응(평형)·ref는 지연(lag) = 부피·시간상수 비대칭이 유력 정체.

- **결론**: V4 평탄 방식은 CO₂ 안정 시 완벽 작동(재테스트 증명). 드리프트 구간만 문제 → **폭기 가스 안정화(스크러버/외기 라인)로 평형 고정**이 robust 해법(CO₂ 안정/드리프트 무관 깨끗한 평탄+차동 상쇄).

### ★드리프트 오차의 두 번째 레버 = 토출량 강화(τ 단축) (사용자 제안 2026-06-18)
드리프트 오차 ≈ **gap(tank↔ref 측정 시간차) × dC/dt**. 단일 프로브 순차라 gap을 0으로 못 만드니 **줄이는 게 정답**. 평형 τ ∝ V/(K_H·**Q_gas**) → **토출량 Q_gas를 키우면 τ 단축 → 평탄 빨리 도달 → 측정 총시간·gap 비례 축소 → 드리프트 오차 비례 감소.** 17:08(gap ~25분 → −0.063) 정통 타격.
- **두 해법 비교(같은 드리프트 문제):** (a) 가스 기준 고정(스크러버/외기라인)=attractor 안 움직임=gap 완전 무관(완전 제거). (b) 시간 단축(강폭기)=attractor 움직일 시간 안 줌=gap 비례 축소(부분). **(b)가 싸고 소모품 없음 → 1차 레버, 부족하면 (a).**
- **건드리지 못하는 것:** ① systematic offset −0.75 dKH(영점보정 몫) ② 증발 흩어짐 ③ **평형 값 자체**(강폭기는 *같은* 대기천장에 *빨리* 갈 뿐, 더 정확한 값 아님 — 과폭기 불가). ⟹ **CO₂ 안정 측정에선 강폭기든 약폭기든 수렴 ΔpH 동일**(깨끗한 3점은 강폭기였어도 같은 값). 이득은 *드리프트 구간 보험*뿐.
- **한계(sweet spot):** kLa가 높은 VVM서 포화(기포합체·gas holdup)→τ 무한정 못 줄임. 부작용=거품·튐·**프로브 미세기포 노이즈**(TRIM_N/CLEAN을 넣은 이유)·에어로졸 증발. 너무 세면 노이즈가 정확도 도로 깎음.
- **검증:** `test_aeration_plateau.py`로 폭기 세기만 올려 재실행(D13 PWM duty↑ 가능 시) → 평탄시간 ~29분→~10분대로 줄고 곡선 여전히 단조·무노이즈면 = 공짜로 드리프트 강건성. 기포 지터 늘면 그게 한계 신호.

### ★레버(a) 채택 = 밀폐(공통 헤드스페이스) — 외기라인은 백업 (사용자 결정 2026-06-18)
가스기준 고정(a)의 구현으로 **외기라인보다 밀폐를 기본 채택.** 원리·이점:
- **원리적으로 더 맞음:** 차동 요구조건은 "절대 pCO₂ 안정"이 아니라 "측정 순간 ref·tank가 *같은* pCO₂"(알칼리니티는 pCO₂ 무관 보존). 밀폐=ref·tank가 **하나의 박스 공기를 공유**하며 외부 차단 → 공통 가스가 *본질적으로 보장* + 방CO₂ 드리프트 원천 차단. 외기라인은 둘 다 외기 흡입해야 하는 간접 방식. reef2reef 사용자 밀폐통 성공("변동 멈춤")=현장 검증.
- **★보너스(외기라인엔 없음): 증발도 잡음.** 밀폐로 증발 급감 → 13:00 +0.29 dKH 증발 confound까지 동시 해결. **새벽 CO₂ + 13:00 증발 두 잔여 오차원을 한 번에 덮음.**
- **★필수 조건:** ref·tank가 **하나의 *공통* 밀폐 헤드스페이스 공유**해야 함. 분리 밀폐(ref=박스A·tank=박스B)는 두 헤드스페이스가 따로 드리프트→상쇄 깨짐. 펌프 흡입을 공통 헤드스페이스서 폐루프로. (현 셋업=5L통 반밀폐+200mL 개방+펌프 *방공기* 흡입 → 통+컵+흡입을 한 밀폐공간으로 묶기.)
- **남는 리스크(대체로 양호):** 박스 CO₂ 느린 드리프트(물 outgassing/온도)—단 **5L 참조수 알칼리니티 버퍼가 헤드스페이스 pCO₂ 핀고정**(방공기보다 훨 안정). 펌프 발열→온도욕조 안정 필요. 응결·순압 점검.
- **검증(한 사이클):** 밀폐 후 V4로 **새벽(고-CO₂) 사이클** 1회 → ① ΔpH 새벽 dip 소멸 = CO₂ 차단 성공 ② 13:00류 증발 흩어짐 감소 = 증발도 잡힘. 둘 다면 밀폐 우위 확정.

## 평탄 판정 = 정수 milli-pH 윈도우 span (float 지터 + 크리프 조기latch 해결, 2026-06-17)

plateau_run2 분석에서 두 결함 발견 → 두 스크립트(test + measure) 모두 수정:
1. **float 비교 지터**: `abs(ph−prev) ≤ 0.001` 는 화면상 "0.001"이어도 float가 0.00100…01(>0.001)이라 True/False가 들쭉날쭉 → 연속 카운터가 리셋돼 702s 아닌 1053s에 latch(설명≠결과의 원인). **정수 milli-pH(`round(ph*1000)`) 비교로 해소.**
2. **연속 Δ는 너무 느슨**: 결정론적(정수)으로 "연속 1mpH ×3"은 느린 지수 꼬리(~1mpH/스텝)에 걸려 **702s/8.053(평형 8.062 −0.009)** 조기 latch. (테스트가 "잘 됐던" 8.062는 지터가 우연히 latch를 늦춘 행운.)
3. **해결 = 윈도우 span**(최근 4개 정수 mpH max−min ≤ 2mpH): 정수라 지터 없고, 4스텝 크리프는 span>2라 안 걸림. run2 CSV 재현 latch=**8.060(평형 −0.002)**. 두 스크립트 동일 기준.

상세 진단·가설 이력은 메모리 `project-kh-offset-aeration-test` 참조.
