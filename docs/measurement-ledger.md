# KH 측정 대장 (코드 상태 · 환경 · 결과)

ΔpH/dKH 검증을 계속하려면 "각 측정이 어떤 코드·환경에서 나왔는가"를 정확히 알아야 한다.
git 커밋만으로는 부족하다 — **6/16 13:00 변경(STABLE·EPS)과 v2 재작성은 커밋되지 않고 작업트리에서만
돌았다.** 그래서 이 대장에 측정 시각별로 직접 기록한다.

**기록 원칙:** 새 측정이 끝날 때마다 아래 표에 한 줄 추가. 값은 `measure_kh.log`(실제 실행 진실)와
`dkh.dat`에서 가져온다. 코드를 바꾸면 ① 새 "코드 버전" 항목을 정의하고 ② 가능하면 **커밋해 SHA를 남긴다**
(미커밋이면 표에 `(작업트리)`로 표기). 환경(방 CO₂)은 직접 로그가 없으므로 **절대 pH를 대리지표**로 쓴다
(절대 pH 낮음 = 방 CO₂ 높음). dKH 계산: `tank_dKH = 8.448 × 10^(−(ref_pH−tank_pH))`, ΔpH는 `tank_pH − ref_pH`.

dkh.dat 컬럼: `HH ref_pH tank_pH ref_kh(앵커 8.448) tank_kh temp`
- **에러 표식**: 5개 값 전부 `0.000…0.0` = 측정 실패/타임아웃/KCl 소크 실패. **에러 래치**(V4): 마지막 줄이 에러 표식이면 다음 실행은 측정 없이 에러 표식만 재기록 — *수동으로 마지막 에러 줄 제거* 전까지(오류 무인 반복측정 방지).
- **평탄 미도달 표식**(V4): `tank_kh`가 **음수(−)** = 정상 측정이나 평형 미도달(값 크기 유효, 서버가 부호로 인지). 에러 표식(전부 0)과 구분되어 래치 안 걸림.

### ★통찰 — 6/16 vs 6/17 raw pH는 우연이 아니다 (2026-06-17)
```
6/16 05:00:  ref 7.810   tank 7.751
6/17 05:00:  ref 7.997   tank 7.939
  Δ(day→day): +0.187      +0.188      ← ① ref·tank 똑같이 상승 = 공통모드
  ΔpH(t−r):   −0.059      −0.058      ← ② 차동은 1 mpH로 동일
```
1. **공통모드 +0.188 → 차동 상쇄.** ref·tank가 정확히 같은 양 이동(전극 드리프트 또는 그날 밤 방 CO₂ 수준차 — 어느 쪽이든 공통모드) → ΔpH 보존. ⟹ **절대 pH는 날짜 간 "방 CO₂ 대리지표"로 부적합(공통모드 혼입). 깨끗한 신호는 ΔpH뿐.** (측정 기록표의 "절대 pH=CO₂ 대리"는 *같은 날 내*에서만 신뢰.)
2. **ΔpH가 코드(V1/V2)·간격(377/313s)·절대수준 다 달라도 1 mpH로 재현 = 안정적·결정론적 계통오차.** ⟹ ① 간격/시퀀스가 원인 아님을 강하게 보강, ② 재현되니 **영점보정 가능한 종류**(랜덤·드리프트 아님).
3. **(가설) ΔpH가 CO₂ 스윙 크기에 비례 안 함 → dC/dt 모델 흔들림.** 밤샘 축적 정황: 6/15 21:00(8.10)→6/16 05:00(7.78) ≈ −0.32(큰 축적) vs 6/16 21:21(8.03)→6/17 05:00(7.97) ≈ −0.06(작은 축적)인데 ΔpH 동일. "ΔpH≈간격×dC/dt"면 6/17(간격 짧음)이 더 작아야 하나 동일 → dC/dt가 마침 20%↑여야 상쇄(부자연). ⟹ **새벽 ΔpH는 rate 기반 과도가 아니라 닫힘(고-CO₂) 조건의 포화/구조적 비대칭** 쪽. (※절대 pH 기반이라 드리프트 혼입 가능, 확정 아닌 가설.)
4. **검정 제안(순수 진단, 해법 아님):** 방을 *스윙 없이 수준만* 고-CO₂로 안정시켜 측정 → 그래도 −0.058이면 "수준 의존 구조 비대칭" 확정(rate 아님).

**해석 (정직 — 이 세 점은 간격 단독 효과를 못 가른다):**
- **6/15 ≡ 6/16: 간격 완전 동일(377s)인데 ΔpH +0.020 vs −0.059.** 간격이 같으니 차이는 순전히 환경(dC/dt, 환기 유무). 깨끗한 대조 → "ΔpH ≈ 간격 × dC/dt"의 dC/dt 항을 입증.
- **6/16 vs 6/17: 간격 377→313s(~17%↓)인데 ΔpH −0.059→−0.058 거의 불변.** 표면상 "간격 줄여도 효과 없음" = **간격이 레버가 아님** 쪽 시사. 단 다른 밤(dC/dt 다를 수 있음)·다른 코드라 간격 단독 비교 불가.
- **결론: 세 점만으론 간격(timing) 단독 효과 분리 불가.** 진짜 검정 = **같은 밤(같은 dC/dt)에 간격/순서만 바꿔 연속 측정.**
- 보조 단서: 세 세션 모두 수렴 2회 연속 **Δ=0.0000(45초 간격)** → ~1분 스케일에선 시료 표류 없음. 분 단위 간격차가 −0.058을 만든다는 그림과 다소 어긋남 → 밀봉/안정가스 가설 쪽에 무게.

## 현재까지의 핵심 관찰 (이 대장이 뒷받침하는 결론)

1. **새벽(05:00) 음 ΔpH는 코드 버전과 무관하게 방 CO₂ 높을 때 나타난다.** V1(6/16 05:00 −0.059)과 V2(6/17 05:00 −0.058)가 동일. v2(동시폭기·시간상수 일치 시도)도 못 잡음.
2. **측정 간격 "크기"는 day-to-day 변수가 아니다(단, 간격은 누출 통로다 — 위 설계 원칙의 정정 참조).** 연속 이틀 새벽 6/15 **+0.020**(V0) vs 6/16 **−0.059**(V1)로 부호까지 반대. V0·V1은 측정 시퀀스·간격(~6.3분) 동일, 6/15는 수동 ron으로 사전폭기까지 동등. 그런데 부호 반전 → 변한 건 간격·폭기상태가 아니라 **방 CO₂ 변화율(dC/dt)**. 정확한 관계 **ΔpH ≈ 간격 × dC/dt**: 간격 일정 → 변동은 dC/dt 몫(6/15 환기 dC/dt≈0, 6/16 닫힘 dC/dt 큼).

2. **연속 Δ는 너무 느슨**: 결정론적(정수)으로 "연속 1mpH ×3"은 느린 지수 꼬리(~1mpH/스텝)에 걸려 **702s/8.053(평형 8.062 −0.009)** 조기 latch. (테스트가 "잘 됐던" 8.062는 지터가 우연히 latch를 늦춘 행운.)
3. **해결 = 윈도우 span**(최근 4개 정수 mpH max−min ≤ 2mpH): 정수라 지터 없고, 4스텝 크리프는 span>2라 안 걸림. run2 CSV 재현 latch=**8.060(평형 −0.002)**. 두 스크립트 동일 기준.

상세 진단·가설 이력은 메모리 `project-kh-offset-aeration-test` 참조.
